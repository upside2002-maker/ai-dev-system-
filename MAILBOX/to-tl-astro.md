## 2026-05-25 (вечер) — STACK UPDATE от Admin, для TL Astro

**Что:** стек этапа 2 окончательно решён — **Yandex Cloud Object Storage + Yandex CDN + Yandex DNS** для хостинга, **Astro framework** для генератора. Cloudflare Pages (упомянутый утром того же дня) снят с рассмотрения.

**Почему смена:** Admin сделал факт-чек через WebSearch — с июня 2025 года Роскомнадзор фактически блокирует Cloudflare через «16-килобайтный занавес». Российские ISP (Ростелеком, МТС, Мегафон, Билайн, МГТС, >90% русской аудитории) ограничивают каждое соединение к CF до 16 КБ. Cloudflare официально не может ничего с этим сделать. Для воронки Марины (88.6% RU аудитория) это смерть конверсии.

**Yandex Cloud в free tier:** 1 ГБ storage + 10 ГБ трафика/мес — хватит с большим запасом для блог-воронки. SSL автоматический (Yandex Certificate Manager). CDN покрывает РФ + ближнее зарубежье. Российская инфраструктура без блокировок.

**Что меняется для BA Astro:**
- В §5 deliverables (tech-рекомендация) уже не выбор стека (он решён), а только выбор **CMS-механизма** для Марины: markdown в git через GitHub web-editor vs Decap CMS vs sanity-free. Сделай рекомендацию с обоснованием.
- В §3 deliverables (архитектура воронки) — учитывай что хостинг российский, CDN российский, SSL Yandex Certificate Manager.
- Остальное (Stylebook, контент-план, пилотный пост, wireframe) — без изменений.

**Что нужно от Owner'а перед этапом 2 (не блокирует этап 1, разрулим позже):**
- Создать аккаунт Yandex Cloud (5 минут, email).
- Привязать карту (free tier требует, списаний нет в лимитах).
- Решить домен: бесплатный `*.website.yandexcloud.net` поддомен или купить нормальный (~500 ₽/год).

ТЗ `marina-content-funnel-spec-2026-05-22.md` обновлено — новый раздел «Почему НЕ Cloudflare Pages» + переписанный «Стек этапа 2» с обоснованием Yandex Cloud.
