# CRM Workflow UI Playbook

Этот playbook нужен для интерфейсов, которыми реально пользуется менеджер в операционке.

## Главный принцип

Интерфейс должен показывать не "всё, что умеет система", а "что делать сейчас".

## Для менеджерского CRM

Хороший интерфейс:

- показывает один текущий next action;
- прячет вторичную информацию до запроса;
- позволяет сделать основной шаг за 1-2 клика;
- визуально разделяет pre-deal, active deal и post-quote режимы;
- не заставляет человека думать о внутренней архитектуре.

Плохой интерфейс:

- вываливает весь pipeline сразу;
- заставляет вручную искать нужный action;
- смешивает sourcing, quote и post-quote в одном равноправном блоке;
- требует помнить, что означает `in_stock`, `quoted`, `sourcing` и т.д.

## Правило для карточки сделки

Карточка должна отвечать на три вопроса:

1. Что уже известно?
2. Что главное сделать прямо сейчас?
3. Что произойдет после этого?

## Рекомендации по структуре

- Левая колонка: входящие сущности и список сделок.
- Центр: основной workflow текущей задачи.
- Правая колонка: контекст и решение по текущей сделке.

## Для Sitka-подобной CRM

Нужно отделять:

- "поиск до сделки";
- "сделка создана и идет подбор";
- "КП готово и нужно решение клиента";
- "послепродажные шаги".

## Что должен подсказывать UI

- можно ли уже создать сделку;
- сколько пригодных офферов найдено;
- есть ли уже готовое КП;
- ждет ли система решения клиента;
- какой один следующий action должен нажать менеджер.
