# `.claude` in `ai-dev-system`

Эта папка не является частью основного knowledge surface проекта.

## Что здесь лежит

- `worktrees/` — служебные копии / форки рабочего дерева для отдельных
  агентных сессий

## Как к этому относиться

- не читать `worktrees/` как источник правды о системе
- не ссылаться на файлы внутри `worktrees/` из onboarding / overlay docs
- не коммитить содержимое `worktrees/` в основной history

Источники правды для `ai-dev-system` остаются в корне репозитория,
`project-overlays/`, `playbooks/`, `corrections/`, `prompts/`,
`evals/`.

## Почему это важно

Без этого пояснения `worktrees/` легко выглядят как часть продукта,
хотя это только рабочий технический слой. Из-за этого агент может
случайно читать устаревшие или дублированные копии документов.
