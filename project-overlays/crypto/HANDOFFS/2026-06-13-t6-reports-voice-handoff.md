## Операторский язык (как переведены термины)

Единый словарь `reports/vocab.py`. Машинная часть (JSON статус-снимка) — для
панели/секретаря-парсера, там коды зон и `data_rights` уместны. Человеческая
часть (две фразы, дневной отчёт, события) — переведена:

| Техтермин ядра | В лицо человеку |
|---|---|
| `data_rights=observe_only` / `critical_ok=false` | «данных для денежных решений нет, наблюдаю» |
| `data_rights=full` / `critical_ok=true` | «данные на месте, могу действовать» |
| зона `hot` / `extreme` | «перегрев» / «крайний перегрев» |
| зона `capitulation` | «паника, всё распродают» |
| зона `cheap` / `neutral` / `warm` | «дёшево» / «спокойно» / «разогрев» |
| атом `TakeProfit` | «отщипнул кусок прибыли» |
| атом `BuyDipFromBuffer` | «докупил на просадке из запаса» |
| атом `ManualReview` | «жду подтверждения от тебя» |
| атом `Observe` / нет денежных атомов | «ничего не предпринимаю» |
| семья `valuation`/`leverage`/`momentum` | «оценка стоимости»/«плечо в рынке»/«импульс» |
| эталон `HODL_BTC` / `FIFTY_FIFTY_DRIFT` | «просто держать биткоин»/«поровну и не трогать» |
| доходность `+0.0612 ×10^8` | «в плюсе на 6.12%» |

## Пример «что по крипте» (две фразы, фактический вывод)

- **Наблюдение:** «По крипте данных для денежных решений нет, наблюдаю; по рынку
  — картина неясна, данных мало. Сейчас ничего не предпринимаю; наш бумажный
  счёт в плюсе на 1.83% за период.»
- **Перегрев:** «По крипте данные на месте, могу действовать; по рынку — крайний
  перегрев. Сейчас ничего не предпринимаю; наш бумажный счёт в плюсе на 5.14% за
  период.»
- **Капитуляция:** «По крипте данные на месте, могу действовать; по рынку —
  паника, всё распродают. Сейчас ничего не предпринимаю; наш бумажный счёт в
  минусе на 5.47% за период.»
- **Действие:** «По крипте данные на месте, могу действовать; по рынку — крайний
  перегрев. Сейчас отщипнул кусок прибыли; наш бумажный счёт в плюсе на 6.12% за
  период.»

## Дневной отчёт (фактический вывод, состояние «действие»)

```
=== ЧТО ПО КРИПТЕ ЗА 2026-04-02 ===

  Картина рынка по семьям:
    цена — перегрев
    оценка стоимости — крайний перегрев

  Что делаем: отщипнул кусок прибыли

  Стоимость счёта против эталонов на этот день:
    наш счёт — 1.06123600
    поровну и не трогать — 1.08782000
    поровну с выравниванием — 1.08781100
    просто держать биткоин — 1.09780000
    просто держать эфир — 1.07784000
  За период наш счёт в плюсе на 6.12%.

  Важно про сравнение с эталонами: наш счёт держит часть денег в запасе вне
  рынка, а эталоны почти целиком в рынке. Поэтому на росте мы отстаём не из-за
  слабой стратегии, а из-за меньшей доли в рынке; на падении держимся лучше
  из-за запаса, а не из-за ядра. Сравнивать цифры в лоб нельзя — это разная
  подверженность риску (известное ограничение Я-4).

  События дня:
    · Рынок по крипте перегрет — это подтверждено. Если есть прибыль, готов
      отщипнуть кусок по правилам.
    · По крипте предлагаю действие: отщипнул кусок прибыли. Это движение
      деньгами — жду твоего подтверждения, сам не исполняю.
```

На состоянии «наблюдение» строка решения — «данных для денежных решений нет —
только наблюдаю, денег не трогаю», а событие — предохранитель. То есть «ничего
не делаем» — тоже строка отчёта.

## Я-4 пометка

`daily.EXPOSURE_DISCLAIMER` — честный дисклеймер асимметрии экспозиции, вшит в
дневной отчёт ВСЕГДА, когда есть сравнение с эталонами. Называет суть Я-4 явно:
наш счёт держит запас вне рынка (~20% буфер), эталоны почти целиком в рынке →
на росте отстаём из-за доли, не из-за ядра; на падении держимся лучше из-за
запаса, не из-за ядра; «сравнивать цифры в лоб нельзя». Текст содержит маркер
«Я-4». В двух фразах «что по крипте» цифра против эталонов сознательно НЕ
называется как «обыграли/проиграли рынок» — там собственное состояние; полное
сравнение с дисклеймером — только в дневном отчёте.

## Diversion

`test_diversion_no_exposure_disclaimer_when_compared`. Воспроизводит суть
упущения отчёта Т-4b: сравнение депо с эталонами БЕЗ пометки асимметрии
экспозиции (что вводило бы Owner'а в заблуждение перед решением о живых деньгах,
мандат §5). Тест утверждает: как только в дневном отчёте есть сравнение («против
эталонов» + «наш счёт»), в нём ОБЯЗАН быть дисклеймер Я-4, и он называет
асимметрию явно («вне рынка»/«в рынке», маркер «Я-4»). Если кто-то уберёт
пометку, оставив сравнение, тест упадёт.

## Отклонения

- **JSON статус-снимка несёт коды (`data_rights`, зоны `hot/extreme`).** Это
  машинный блок для панели/секретаря-парсера (§3 — машинный контракт), не
  человеческая фраза. Человеческая часть (две фразы, дневной отчёт, события) —
  полностью переведена и проверена грепом. Контракт сознательно делит вывод на
  два адресата (машинный §3 + человеческий операторский), как и требует память
  системы про «отдельный блок для исполнителя».
- **«Позиции» в статус-снимке = цены дня по активам**, агрегат стоимости —
  отдельным полем `depo_value`. Прогон (Т-4b) не возит по-активную рыночную
  стоимость в `DayResult` (только агрегат `cpds_value`); по-активную стоимость
  по лотам отсюда честно не собрать без дублирования логики переоценки, а её
  трогать нельзя. Поэтому дал цены дня (факт рынка) + общий `depo_value`. Если
  ревью нужна по-активная стоимость позиций — это маленькое расширение DayResult
  в Т-4b, не в отчётах.

## Риски для ревью

1. **Порог «нужен человек».** В v1 «нужен человек» = любое денежное действие дня
   (по мандату §5 живые деньги — только после подтверждения Owner'а). Отдельного
   числового порога «действие больше X%» здесь нет — его несёт ядро
   (`human_review_required`, лимит `max_unconfirmed_action_pct` профиля, Я-2).
   Отчёт агрегирует факт «деньги двигались», а не пере-считывает порог. Если
   ревью хочет, чтобы событие «действие больше порога» опиралось на
   `human_review_required` ядра (а не на факт money_moved прогона) — это
   точечная правка в `events.events_for_day` (поле в DayResult пока не
   прокинуто из решения, money_moved — ближайший доступный сигнал).
2. **«Две фразы» определяются разбиением по `.!?`.** Тест считает фразы
   регуляркой; внутри фраз нет точек-сокращений (проценты пишутся «6.12%» —
   точка между цифр, регулярка `(?<=[.!?])\s+` не режет по ней, т.к. дальше не
   пробел+заглавная — проверено). Если формулировки будут менять — держать
   ровно две завершающие точки.
3. **Зоны рынка для «настроения»/событий берутся по ведущей семье** (оценка →
   цена → импульс → плечо). Это упрощение «настроение рынка одним словом»;
   многосемейный конфликт (оценка hot, импульс cheap) сводится к ведущей.
   Сознательно — для двух фраз и короткого события; дневной отчёт показывает ВСЕ
   семьи по-человечески, конфликт там виден.
4. **Зависимость от формата причины observe_only из Т-4b.** `_reason_human`
   переводит строку причины прогона; если Т-4b сменит текст причины — перевод
   деградирует к общей формулировке (не падает, техтермины не протекают — тест
   греп это страхует).
