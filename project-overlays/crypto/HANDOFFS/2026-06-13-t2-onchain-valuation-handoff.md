## Сделано (чем закрыт каждый блокер)

### Б1 — граница детектора скачков
`collectors/onchain.py` `FailureDetector._is_jump`: строгое `>` заменено на
нестрогое `>=` в ОБЕИХ ветках (нулевая и ненулевая база). Значение РОВНО на
пороге теперь выпадает (метрика → None → семья молчит). Закрыта дыра
предохранителя у нулевой базы: при совпадении мусора по всем трём метрикам
valuation у нулевой базы больше не пройдёт ложный critical_ok=true.

### Б2 — единая наблюдаемость сбоев он-чейн (рецидив Т-1)
Убраны все молчаливые except и молчаливый возврат `{}`. Введены типы-итоги с
причиной:
- `BrowserRead(metrics, degraded, transient)` — итог браузерного чтения;
- `HistoryRead(series, degraded)` — итог чтения рядов Coin Metrics.

`read_browser_page` различает классы сбоя и НАЗЫВАЕТ каждый:
- пустой/нет HTML → `degraded="пустой ответ (нет HTML)"`, источник недоступен;
- битый HTML (парсер упал) → `degraded="нечитаемый HTML: ..."`, недоступен;
- нет таблицы метрик (анти-бот 403/редирект) → `degraded="нет таблицы метрик..."`, недоступен;
- таблица есть, метрики не распознаны (смена вёрстки) → `degraded="смена вёрстки..."`, `transient=True`.

`fetch_browser_read` оборачивает 403/таймаут/обрыв (FetchError) в `degraded` (раньше молча `{}`).
`fetch_coinmetrics_history` теперь возвращает `HistoryRead`: сбой сети → причина;
ПУСТОЙ ответ (нет рядов) → отдельная причина `"пустой ответ (нет рядов истории)"`
— отличается от деградации сети.

Различение «временно/смена вёрстки» (`transient=True`) от «источник упал»
(`transient=False`) — для будущей политики ретраев; на предохранитель не влияет
(любой degraded → семья молчит).

CLI (`cli.py` `cmd_snapshot`): единый механизм `DegradeLog` (выделен из бывшей
локальной замыкашки `_degrade` в модульный класс — переиспользуется тем же путём,
что farside/defillama Т-1). Каждый класс сбоя он-чейн → строка в лог
«источник X недоступен: причина» И `source:<имя>` в `data_quality.missing`. Дополнительно:
метрики пришли, но все снёс детектор (вне-диапазона/скачок) → отдельная пометка
«метрики не прошли детектор сбоев».

Совместимость: старые `parse_browser_page`/`fetch_browser_metrics` оставлены
тонкими обёртками (`.metrics`) — прежние зелёные тесты Т-2 не сломаны.

### Б3 — документировать предположение

## Отклонения

- Граница `>=` применена и к НЕнулевой базе (не только к нулевой, как буквально в
  Б1): значение ровно на относительном пороге тоже выпадает. Это консервативнее в
  сторону предохранителя, не ослабляет его; прежние зелёные тесты целы.
- Бывшая локальная замыкашка `_degrade` поднята в модульный класс `DegradeLog`
  (тот же механизм, теперь тестируемый и явно переиспользуемый) — чтобы Б2 опирался
  на ОДИН учёт деградации, а не плодил параллельный.

## Риски для ревью

  честной деградацией; реальная подстановка живого браузер-инструмента вне Т-2.
- Дыра Д-1 (нет глубокой истории он-чейн) не закрывалась — вне границ доработки
