## Б1 — утечка ключа FRED — ЗАКРЫТ

Проверял руками тремя путями, с заданным `FRED_API_KEY`:
- **`redact_url` напрямую** (несколько параметров): на URL с `series_id`,
  `file_type`, `api_key`, `sort_order`, `limit` все ЗНАЧЕНИЯ → `***`, хост и путь
  видны. На URL с одним `api_key` и на URL без query — корректно.
- **Реальный путь `fetch_gold → get_json → get_text → urlopen`**: смоделировал
  (а) сетевой сбой `URLError` и (б) ответ-не-JSON. В обоих случаях секрет
  отсутствует и в тексте `FetchError`, и в полном трейсбеке (`__cause__`); на
  месте значения — `api_key=***`, хост `api.stlouisfed.org` виден.
- **Через CLI `cmd_snapshot`** при заданном ключе и упавшем FRED: ключа НЕТ ни в
  stdout, ни в stderr; строка деградации печатает `api_key=***`.

**Греп всех коллекторов:** все URL идут через единственную точку
`http.get_text/get_json`, где маскирует `redact_url`. Ни один коллектор не строит
свой `raise`/`print` с сырым URL. FRED — единственный источник с секретом;
даже query binance/coinbase/defillama маскируются. `farside` ловит `FetchError`
и возвращает None (URL не светит). Литералов ключей в репо нет (`git grep`).

Конструировал истории и звал `build_snapshot` напрямую. Порог `max_staleness_days`
берётся из конфига (`config/snapshot.yaml: 2`), не зашит:
- **просрочено** (последняя точка 2026-06-01, снимок 2026-06-12, возраст 11д) →

- **граница** (возраст ровно 2 == порог) → НЕ stale (сравнение строгое `>`);
  возраст 3 → stale. Поведение чёткое.
- **нет метки времени** (`last_close_ms=None`) → проверка свежести пропущена,
  никаких `asset:`/`family:` меток.
