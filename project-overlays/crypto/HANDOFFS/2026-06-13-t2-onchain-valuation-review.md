# Ревью Т-2 (он-чейн, критичная семья) — сведённый вердикт Admin'а

**Дата:** 2026-06-13
**Метод:** независимое многолинзовое ревью (5 линз + верификация), ветка
feat/t-2-onchain, изолированно от Т-4a.
**Сырой итог:** REJECT, 2 критических + 3 средних.
**Вердикт Admin'а: ДОРАБОТКА.** Предохранитель ядра ЦЕЛ (деньги не текут);
блокеры — граница детектора и наблюдаемость (рецидив урока Т-1).

## Разбор находок

- **«Сборка не ОК» у линз — ЛОЖНОЕ.** Проверил руками: `make check` на
  feat/t-2-onchain зелёный. У линз нет сети под установку зависимостей.
- **КРИТ-1 (граница детектора скачков): реальна, чинить.** `_is_jump` при
  базе ~0 использует строгое `> max_rel_jump`; значение ровно на пороге
  проходит как «не скачок». Узко (нужно совпадение по всем трём метрикам
  valuation у нулевой базы → ложный critical_ok=true на мусоре), но это
  предохранитель денег — нестрогую границу не оставляем. Правка: `>=` +
  тест на ровно граничное значение.
- **КРИТ-2 (молчаливый сбой источника): по сути блокер НАБЛЮДАЕМОСТИ, не
  денежной утечки.** `fetch_browser_metrics` ловит FetchError (403 анти-бот)
  молча → возвращает {}, `_degrade` не зовётся, оператор не видит реальную
  причину. Предохранитель ЦЕЛ (семья молчит → critical_ok=false → деньги
  не текут). НО это прямой РЕЦИДИВ урока Т-1 (Б2: единая наблюдаемость
  деградации) — параллельная сессия его не несла. Для критичной семьи
  оператор обязан видеть, почему она замолчала. Чинить.
- **СРЕД (кластер наблюдаемости + документирование):** молчаливый except в
  `parse_browser_page`; пустой ответ Coin Metrics не отличается от
  деградации; хрупкое недокументированное предположение `max_rel_jump<1`
  (абсолютный порог при нулевой базе). Всё в ту же доработку.

## Блокеры доработки (ветка feat/t-2-onchain)

- **Б1 — граница детектора:** `_is_jump` при нулевой базе → `>=` (значение
  ровно на пороге выпадает); тест на граничное значение (ровно
  max_rel_jump от нуля → метрика выпадает → семья честно молчит).
- **Б2 — единая наблюдаемость сбоев он-чейн (рецидив Т-1):** ЛЮБОЙ сбой
  источника (403/таймаут/битый HTML/пустой Coin Metrics/вне-диапазона) →
  явная строка в лог И пометка degraded/missing (как farside/Т-1).
  Молчаливые except и молчаливый возврат {} запрещены. Где возможно —
  различать «временно (смена вёрстки)» от «источник упал». Тесты на каждый
  класс сбоя: лог есть + пометка есть + critical_ok=false.
- **Б3 — документировать предположение:** `max_rel_jump` при нулевой базе
  используется как АБСОЛЮТНЫЙ порог; зафиксировать в коде и config/snapshot.yaml
  (предупредить будущую калибровку Т-7, чтобы не сломать детектор).

## Что устояло

Предохранитель главного инварианта (сбой → семья молчит → critical_ok=false
→ ядро observe_only) РАБОТАЕТ на проверенных классах сбоя. Схема
market_snapshot не тронута. Секретов в репо нет. NaN/Inf отсекаются
(`_coerce_finite`). Зоны по квантилям из конфига. `make check` зелёный.
