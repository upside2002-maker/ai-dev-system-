## Problem

Отчётный слой и доклад: статус-снимок крипто-контура для секретаря
(«что по крипте» — две фразы) + утренний дневной отчёт + событийные
уведомления. Над журналами полигона (Т-4a) и прогоном (Т-4b) строит
человекочитаемую сводку и контракт статус-снимка федерации §3.
Архитектура §11 (панель и доклады — текстовый отчёт, НЕ веб-панель),
§3 контракт федерации.

## Контракт (зафиксировать точно)

- **Статус-снимок** по контракту федерации §3: позиции (бумажные), результат
  за период, текущие зоны семей, последние решения, где нужен человек —
  компактный JSON + текстовая выжимка. Читается секретарём для ответа
  «что по крипте» в две фразы (операторский язык, без техжаргона).
- **Дневной отчёт:** одна страница — зоны семей дня, решение ядра (или
  observe_only с причиной), стоимость депо против эталонов, события.
  «Ничего не делаем» — тоже строка. ОБЯЗАТЕЛЬНО: пометка асимметрии
  экспозиции (Я-4) при сравнении с эталонами — честный дисклеймер, чтобы
  статистика бумаги не вводила в заблуждение.
- **Событийные уведомления:** перегрев/капитуляция подтверждены, действие
  больше порога (нужно подтверждение), сработал предохранитель (семья
  молчит) — короткий текст для Телеграма/голоса.
- Операторский язык: человеку — без техжаргона (никаких critical_ok,
  observe_only в лицо — «данных нет, наблюдаю»); техдетали в отдельный
  блок «для исполнителя», если нужны.
