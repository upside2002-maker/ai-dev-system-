## Problem

Первый кусок бумажного полигона (Т-4): виртуальный исполнитель. Применяет
решение ядра (DecisionOutput) к состоянию виртуального счёта и обновляет
четыре журнала — те самые, что ядро на следующий день читает как вход.
Замыкает цикл «снимок → ядро → решение → исполнение на бумаге → журналы →
снова вход ядра». Живых денег нет: депо виртуальное, котировки реальные,
издержки моделируются. Прогон по истории, эталоны и отчёт — это Т-4b,
сюда НЕ тащить. Архитектура §8 (полигон), §7 (модель издержек), контракт
ядра из ревью Т-3d (ядро отдаёт ДОЛИ карты, суммы считает обвязка).

## Контракт исполнения (зафиксировать точно)

- Вход исполнителя: текущее состояние счёта (4 журнала + буфер/корзины) +
  DecisionOutput ядра + цена снимка (фактическая цена исполнения в симуляции).
- `TakeProfit` (доля position_pct позиции актива): продать долю по цене
  снимка с поправкой на издержки → **realized_net_profit ПО ФАКТУ** =
  proceeds(с издержками) − пропорц. себестоимость (средневзвешенная из лотов).
  Запись в realized_profit_journal; лоты уменьшаются (status partial/closed).
- `DistributeProfit`: разложить realized_net_profit по корзинам, умножая
  ДОЛИ из distribution_plan решения на realized (контракт Т-3d: ядро дало
  доли, полигон умножает на ФАКТ). Σ сумм = realized до единицы (остаток —
  тем же правилом, что в ядре: крупнейшей доле). Запись в distribution_journal.
- `CreateSaleLock`: записать блокировку (sale_price, qty, reset_rules) в
  sale_lock_journal, status active.
- `BuyDipFromBuffer`: купить из буфера долей на цену снимка с издержками →
  новый лот (origin=dip_buy), буфер уменьшается.
- `Observe` / `ManualReview`: состояние не меняется (ManualReview — пометка
  «ждёт человека», в симуляции трактуется как Observe, фиксируется в логе).
- Издержки (§7): комиссия (мейкер/тейкер) + спред + проскальзывание —
  константы из конфига полигона. Денежная арифметика — на фикс. точности
  платформы (переиспользовать numfmt/canonical из принятого Т-1), БЕЗ float.
- Детерминизм: одно и то же (состояние, решение, цена, конфиг) → один
  результат; журналы валидны по схемам.
