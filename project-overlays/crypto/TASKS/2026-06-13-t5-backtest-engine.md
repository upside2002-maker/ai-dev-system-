## Problem

Бэктест-движок по-взрослому: прогон гипотез на истории со строгим
запретом заглядывания в будущее (point-in-time) и проверкой вне выборки
(walk-forward). Т-4b даёт прогон одной стратегии день-за-днём; Т-5 строит
над ним дисциплину честного бэктеста: окна калибровки/проверки, метрики
с издержками и штормовыми окнами, контракт стратегии (StrategySpec →
BacktestResult). Архитектура §6 (труба стратегии), §7 (бэктест: point-in-
time, walk-forward, издержки, штормовые окна).

## Контракт (зафиксировать точно)

- **Point-in-time строго:** на дату решения стратегия видит ТОЛЬКО данные,
  известные на эту дату (никаких будущих свечей/метрик/ревизий). Это
  главный инвариант — нарушение = бэктест врёт.
- **Walk-forward:** калибровка параметров на окне N, проверка на следующем
  окне M (out-of-sample); скольжение по истории. Параметры калибруются
  ТОЛЬКО на калибровочном окне, метрики считаются на проверочном.
- **BacktestResult** (по §6): период, окна walk-forward, метрики (чистая
  доходность после издержек, макс. просадка, оборот, уплаченные комиссии,
  сравнение с бенчмарками BTC/ETH/50-50/CPDS-база), поведение в штормовые
  окна (март 2020, медвежий 2022, крах FTX ноябрь 2022 — ОБЯЗАТЕЛЬНЫЕ окна),
  пробелы данных, хэш результата (воспроизводимость).
- **StrategySpec** (минимальный для v1): id, гипотеза (откуда деньги —
  обязательное поле), активы, требуемые данные, частота решений, правила
  (исполнимы без модели), параметры, класс риска. Для v1 достаточно
  обернуть существующую CPDS-стратегию (прогон Т-4b) в этот контракт +
  каркас для будущих Г-1/Г-2/Г-3.
- Издержки — модель §7 (переиспользовать config/paper.yaml). Деньги —
  фикс. точность, без float.

## ЧЕСТНАЯ РАМКА (обязательно)

Полная валидация CPDS на истории упирается в Д-1 (нет глубокой истории
он-чейн) — на исторических окнах семья valuation чаще молчит, действий
мало. Т-5 строит ДВИЖОК; полнота бэктеста CPDS ограничена Д-1. Движок
валиден для (а) инфраструктуры walk-forward/point-in-time, (б) стратегий
без он-чейн (Г-1 кэрри, Г-3 тренд — позже), (в) накопления вперёд. Где
данных нет — честно помечать, не выдумывать. Асимметрия экспозиции (Я-4)
тоже применима к сравнению с бенчмарками — учесть.
