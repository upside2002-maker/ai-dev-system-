## Problem

Второй кусок бумажного полигона: прогон по истории. День за днём собирает
снимок из накопленной истории, подаёт в ядро (cpds-core), применяет решение
виртуальным исполнителем (Т-4a, apply_decision), переоценивает депо, ведёт
эталоны «купи-и-держи» и пишет дневной отчёт. Это машина, которая покажет,
чего стоят советы ядра — на бумаге, до живых денег. Архитектура §8 (полигон,
бенчмарки, ежедневная переоценка), §2 (частоты). Работает на master (Т-1
данные + Т-2 он-чейн + Т-3 ядро + Т-4a исполнитель уже влиты).

## Контракт прогона (зафиксировать точно)

- Прогон по диапазону дат: для каждого дня — снимок (из истории Parquet) →
  cpds-core → DecisionOutput → apply_decision (Т-4a) → обновлённые журналы +
  депо → переоценка. Детерминированно, воспроизводимо.
- Эталоны: купи-и-держи BTC, купи-и-держи ETH, 50/50 (ежедневная
  ребалансировка или дрейф — зафиксировать какой; рекомендую дрейф +
  отдельно ребаланс), и действующая CPDS-стратегия. Все на одном стартовом
  капитале, одной модели издержек.
- Переоценка депо: стоимость = Σ(лоты × цена дня) + буфер + корзины
  (терминал по стоимости вывода, не по рынку — он выведен). Денежная
  арифметика — фикс. точность, без float.
- Дневной отчёт: одна строка/блок на день — зоны семей, решение ядра
  (или observe_only), стоимость депо, стоимость эталонов, отметки событий.
  «Ничего не делаем» — тоже строка.
- Сводка прогона: итоговая доходность, макс. просадка, оборот, число
  действий, сравнение с эталонами.

## ЧЕСТНАЯ РАМКА (обязательно отразить)

Полноценная статистика CPDS на истории упирается в дыру Д-1 (глубокой
истории он-чейн нет): на исторических днях семья valuation чаще молчит →
critical_ok=false → ядро в observe_only → действий мало. Это НЕ дефект
прогона. Прогон валиден для (а) инфраструктуры день-за-днём, (б) стратегий
без он-чейн зависимости (кэрри/тренд позже), (в) накопления вперёд. Где
он-чейн истории нет — отчёт честно помечает «observe_only: нет valuation»,
НЕ выдумывает данные. Зафиксировать в отчёте и HANDOFF; при нужде —
строку в KNOWN_LIMITATIONS.
