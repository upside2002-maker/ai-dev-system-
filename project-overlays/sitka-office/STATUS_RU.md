Мелкие задачи, не срочные, копятся в `OPERATING/backlog.md`. Самые ближайшие:

- Исправить значение по умолчанию `localhost:8080` в фронтенд-сборке (3-5 строк, класс C).
- Гонка курсора Avito-опросника при холодном старте — пропускает окно ~60 секунд один раз (10-15 строк, класс A на стороне ядра).
- Зеркалирование серверных правок `docker-compose.override.yml` обратно в репозиторий (~30 строк, класс C), чтобы следующий деплой не повторял их вручную.
- Чёрновая задача `TASKS/2026-04-29-dm7-c-backend-widget-prereq.md` — расширение `Api.TransactionResp` для будущей разбивки кассы по типам расходов. Не срочна, нужна только когда оператор реально захочет разбивку. **Отложенный платёж по новой системе режимов:** задача помечена как класс A, но в шапке нет поля `Mode:`. Перед возвращением её в работу нужно будет дописать `Mode: strict` (или явно понизить класс с обоснованием в разделе «Контекст»). До этого — задача стоит черновиком; если попытаться её принять сейчас, проверка на стороне скрипта `accept-task` откажет с понятным сообщением.
