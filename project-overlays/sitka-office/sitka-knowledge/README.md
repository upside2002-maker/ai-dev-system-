# Sitka Knowledge Base

База знаний по моделям Sitka для product-assistant бота и для справки оператора.

## Структура

```
sitka-knowledge/
  README.md              # этот файл
  catalog.tsv            # мастер-список всех мужских моделей Sitka (~281 шт)
  models/                # markdown per model
    jetstream-jacket.md  # full-tier (развёрнутая)
    minimal-*.md         # minimal-tier (структурная)
    ...
```

## Два уровня карточек (tiers)

### Full tier — для ключевых моделей (топ-20-30)

Развёрнутая карточка со всеми секциями: описание, для каких условий, отличия от похожих моделей, характерные вопросы клиентов, технические детали, **внешние обзоры с цитатами и ссылками**, заметки оператора.

Размер: ~5-6KB. Используется для моделей, которые часто обсуждаются с клиентами и требуют нюансированного ответа.

Сейчас full tier: `jetstream-jacket`, `timberline-pant`, `stratus-jacket`, `stratus-pant`, `traverse-jacket`, `traverse-hoody`, `dew-point-jacket`, `dew-point-pant`, `kelvin-aerolite-jacket`.

### Minimal tier — для массового каталога

Структурная карточка: YAML front-matter (все поля как в full) + краткое описание (2-3 предложения) + ссылка на источник. Без развёрнутых секций.

Размер: ~1-1.5KB. Используется для всех остальных моделей каталога. Агент/оператор дописывают развёрнутый контент по мере необходимости — это превращает minimal в full.

### Миграция minimal → full

Когда модель становится часто востребованной:
1. Агент собирает внешние обзоры (Rokslide, GearJunkie, Ron Spomer и т.п.)
2. Добавляет секции: «Для каких условий», «Отличия от похожих моделей», «Характерные вопросы клиентов», «Внешние обзоры»
3. Фиксирует `last_verified`

## catalog.tsv — мастер-индекс

`catalog.tsv` — tab-separated файл со всеми мужскими моделями Sitka после фильтрации (women's / kids / care products исключены). Колонки: `slug`, `name`, `category`, `url`.

281 модель (на 2026-04-20). Открывается в Excel/Numbers/Google Sheets — удобно для просмотра всего каталога, сортировки по категории, поиска.

## Формат файла модели

Каждый файл — markdown с YAML front-matter. Front-matter содержит структурированные поля (которые потом идут в Postgres), тело файла — описательные секции на русском.

### Обязательные поля front-matter

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | string | Официальное название с сайта Sitka |
| `slug` | string | URL-slug, обычно `<name-in-lowercase-with-dashes>` |
| `category` | enum | `jacket` / `pant` / `midlayer` / `hoody` / `glove` / `hat` / `bib` |
| `layer` | enum / null | `l1`–`l5` по Sitka layer system. `null` если не указано и уверенности нет |
| `season` | array | `spring` / `summer` / `early-fall` / `fall` / `late-fall` / `winter` |
| `weight_g` | integer | Вес в граммах, размер L, для штанов — 34 waist |
| `fit` | enum | `athletic` / `standard` / `roomy` / `tapered` |
| `shell` | string \| null | Точная технология оболочки (цитата с сайта) |
| `insulation_g` | integer | Граммаж утеплителя. `0` если без утеплителя |
| `insulation_type` | string \| null | `primaloft` / `down` / `synthetic` / `fleece-lining` / `null` |
| `windproof` | bool \| null | `null` если не указано явно на странице |
| `rain_protection` | enum | `none` / `dwr` / `rain_shell` / `rain_insulated` |
| `breathability` | enum | `low` / `mid` / `high` |
| `comfort_temp_c` | string \| null | Примерный диапазон, например `"+5..+15"` или `null` |
| `use_case` | array | Сценарии применения (на русском) |
| `not_for` | array | Для каких условий **НЕ** подходит (на русском) |
| `tags` | array | Ключевые слова для поиска (на русском и английском) |
| `source_urls` | array | URL страниц sitkagear.com |
| `last_verified` | date | Дата ручной проверки (YYYY-MM-DD) |
| `updated_at` | date | Дата последнего изменения файла |

### Тело файла (на русском)

Обязательные секции:

1. **Короткое описание** — 2-3 предложения, суть модели
2. **Для каких условий** — расширенное описание типичного применения
3. **Отличия от похожих моделей** — ссылки на другие карточки по slug
4. **Характерные вопросы клиентов** — Q&A из реальных переписок
5. **Технические детали** — конструкция, фурнитура, особенности
6. **Источники** — список проверенных URL + дата ручной проверки

## Правила обновления

1. **Структурные поля** (front-matter) — только то, что **явно** указано на sitkagear.com. Если поле не упомянуто — `null` или пропуск, **не гадать**.
2. **Поле `last_verified`** — дата последней сверки с сайтом. Обновляем раз в сезон или при рестайлинге модели.
3. **Секция «Характерные вопросы клиентов»** — писать **только** на основе реальных переписок из Avito/Telegram, не из головы.
4. **«Отличия от похожих моделей»** — актуализировать, когда появляется новая модель в том же классе.

## Anti-confabulation

Не придумывать числа и факты. Если на sitkagear.com не указан граммаж утеплителя — `insulation_g: 0` если модель не утепляется, или `null` с комментарием `# не указано на странице, требует проверки` если есть основания предположить что утеплитель всё же есть.

## Источники данных

- **Первичный**: https://www.sitkagear.com/ (официальный сайт)
- **Вторичный**: переписки клиентов с оператором (для «Характерные вопросы»)
- **Не используется**: сторонние обзоры, блоги, форумы без указания автора
