# ТЗ: модуль профориентации для Astro (метод К. Дарагана)

**Заказчик:** Owner (для астролога Марины). **Поставил:** Admin/Architect.
**Тип:** новый эпик в продукте Astro. **Объём первой версии:** движок + базовое
(черновое) толкование. Полное толкование в голос Марины — следующая фаза.

## Источник (читать обязательно — делаем ПО МЕТОДУ, не «по мотивам»)

Константин Дараган, «Профессиональная астрология» (Мир Урании, 2019). Worker
читает страницы перед реализацией:
- **282–300** — сам алгоритм профориентации и «призвание»;
- **206–209** — планеты как сигнификаторы в натальной карте;

Если что-то в этом ТЗ расходится с книгой — верить книге и отметить расхождение.

## Суть метода (кратко, для ориентира — детали в книге)

Главная идея: любая профессия сводится к **четырём способам получать деньги от
людей**, и каждый завязан на свой дом:
- **X дом** — успех/признание в обществе, руководство, создание своего дела;
- **II дом** — работа на себя, личное предпринимательство (доход ограничен личным трудом);
- **VI дом** — работа по найму (контракт, фиксированный доход);
- **VIII дом** — доход с вложений/инвестиций (рискованно; учитывать только при очень удачных сочетаниях).

Алгоритм «поиск оптимальной области профессиональной деятельности» — пять стадий:

1. **Реализация.** Для каждого из домов X, II, VI, VIII выписать планеты:
   (а) управитель дома; (б) планеты в доме; (в) планеты в соединении или
   гармоничном аспекте с управителем дома; (г) планеты во взаимной рецепции по
   обители/экзальтации с управителем или планетами дома.
2. **Способности.** Выписать сильные планеты человека: (а) управитель Асцендента
   и планеты в I доме; (б) дорифорий и возничий (спутники Солнца); (в) планеты в
   обители/экзальтации; (г) управитель солнечного знака; (д) планеты в соединении
   или гармоничном аспекте с Солнцем; (е) планета в элевации (выше всех над
   горизонтом); (ж) «ручка корзины» / «праща» в фигурах Джонса; (з) планеты на
   благоприятных неподвижных звёздах. Планеты, упомянутые несколько раз, —
   особо значимы.
3. **Фильтрация.** Оставляем планеты, попавшие в ОБА списка (и способность, и
   реализация), у которых строки плотно заполнены. Среди «способностей» приоритет
   — лучшее положение в знаке + лучшая аспектация. Среди «реализации» иерархия:
   угловой (X) > последующий (II) > падающий (VI). VIII — только при очень удачных
   сочетаниях.
4. **Выбор.** Берём топ 2–3 сочетания «планета + дом», в приоритете связанные с X
   домом (если лучшая планета связана с X — это, как правило, оптимальный вариант).
5. **Толкование.** Перевод сочетания на язык клиента: «управитель X в V доме» →
   успех в сфере V дома; планета сужает (Марс → спорт/соревнование, Венера →
   искусство/сцена), знак сужает дальше (Марс в водном знаке → водный спорт /
   море / профессии у воды).

Отдельно — **«призвание»** (стр. 298–300): есть не у всех; видно прямо по карте,
когда профессия и личность нераздельны. Не навязывать миф «у каждого есть призвание».

## Что строим (первая версия = вариант 1)

**Движок** (считает факторы, строит таблицу стадий 1–4, ранжирует, выдаёт топ-2–3
сочетания) **+ базовое толкование** (черновой текст по сочетанию планета+дом+знак).
Полное толкование под все сочетания в голос Марины — следующая фаза, НЕ в этой.

## Архитектура (границы Astro — соблюдать строго)

- **Вся математика и отбор — в ядре (Haskell).** Факторы, таблица, ранжирование,
  выбор вариантов — это домен-математика, ей место в `Domain.*`/ядре, НЕ в Python
  (bright line #7; см. находку AST-ARCH-1 из `docs/ARCHITECT_REVIEW_2026-06-05.md`
  — значимость-логику в Python больше не плодим).
- **Толкование — правила-данные** (`packages/rulesets/` или поле в схеме фактов),
  которые **Марина правит сама**, без программиста. Базовый набор правил —
  черновой, помечен как «правится Мариной».
- **Выдача** — отдельный раздел консультации (и в PDF, и в UI-панели), по образцу
  уже сделанных «возвратов планет» и транзит-календаря. Reuse существующего
  пайплайна сборки консультации.

## Фазы (по образцу эпика «возвраты планет»)

- **Phase A — ядро:** факторы (стадии 1–4) + контракт/схема фактов профориентации
  + golden-тесты на реальных картах (математика заякорена). Tier A (домен-математика)
  → strict + независимый Reviewer (теперь это гейт, см. Correction 021).
- **Phase B — толкование + выдача:** стадия 5 как правила-данные (черновой набор) +
  раздел в консультации (PDF + UI). Tier B.
- (Полное толкование Марины — отдельный трек после, не в этом эпике.)

## Переиспользовать / достроить

Многое уже в ядре (дома, управители, аспекты, достоинства/обитель/экзальтация,
элевация — проверить наличие). **Достроить:** спутники Солнца (дорифорий/возничий),
фигуры Джонса («ручка корзины»/«праща»), список благоприятных неподвижных звёзд,
саму таблицу профориентации (4 дома × 4 признака), ранжирование, базовые правила
толкования.

## Definition of Done (первая версия)

1. По натальной карте модуль выдаёт **2–3 лучших профессиональных сочетания**
   (планета + дом + знак) с **черновым толкованием**.
2. Математика и отбор — **в ядре**; толкование — в **редактируемых правилах-данных**.
3. Выдача — **раздел в консультации** (PDF + UI), как у возвратов/транзитов.
4. **Golden-тесты** на 2–3 реальных картах (математика воспроизводима); соответствие
   методу проверяемо по страницам книги (в HANDOFF — ссылки на страницы).
5. Честно: **полное толкование в голос Марины — следующая фаза**, не выдавать
   черновой текст за финальный.

## Чего НЕ делать

- Не зашивать толкование и веса в Python (bright line #7) — только в ядро/правила.
- Не выдумывать факторы вне метода Дарагана. Сомнение — сверяться с книгой.
- Не обещать «призвание» там, где его в карте нет.
