Проект — **два продукта с разными ЦА:**

- **B2B SaaS** — автоматизация расчётов для других астрологов, перспектива 6-12 мес. Возможен bundle «софт + курсы обучения».

## Ключевые выводы по блокам

### A — Рынок

**B2C.**
- Рынок эзотерики РФ: 100 млрд — 2 трлн ₽ (оценка 2023), рост 20%/год. Применимость к астрологии — не дифференцирована. Используется как сигнал «рынок большой», не как опорная цифра.
- Лидеры отрасли (Мария Штерн, Лиза Васина, Павел Андреев) **комбинируют консультации + школу**. «Только консультации» — редкая конфигурация у лидеров.
- Средний чек: 3-5 k ₽ медиана, топ до 35 k ₽/сессия. Курсы от 15 k ₽ до 490 k ₽ (VIP).
- Позиционирование Марины (5 лет + узкие субниши) — средний+ сегмент с обоснованным чеком 5-15 k ₽/сессия.
- 23% россиян верят в астрологию — работа с доверием критична в продукте.

**B2B SaaS.**
- Ключевой русскоязычный конкурент — **Chronos.mg** (550/2000/10 001 ₽/мес), **часть экосистемы школы LabLife (Павел Андреев)**. Паттерн «курсы + софт» в нише **уже работает**.
- Другие: ZET (desktop pro), Астронова, Gadanis.
- Международные: Solar Fire, AstroSeek, Astro.com, Chani/Sanctuary ($20/мес). Pro-ниша не закрыта для русскоязычных.
- Барьеры входа НЕ технические (эфемериды open source). Барьеры: доверие, сообщество, русскоязычная библиотека интерпретаций.

### B — Юр/фин модель

**B2C.**
- **Самозанятый (НПД)** — рабочее начало до 2.4 M ₽/год. Триггер перехода на ИП УСН 6% — приближение к лимиту ИЛИ запуск курсов с юрлицами ИЛИ старт SaaS.
- **ОКВЭД 96.09** «прочие персональные услуги» — стандарт для астролога-ИП. Для самозанятого — выбор виды деятельности в «Мой налог», без ОКВЭД.
- **ФЗЗПП применим.** Корневая защита — **формулировка услуги как «интерпретация методом X», а не «предсказание результата»**. Это одновременно защищает от ФЗЗПП-претензий и претензий регулятора.
- **Законопроект № 901048-8 о запрете рекламы эзотерики** — внесён апр 2025, правительство НЕ поддержало июль 2025. Статус «висит», политический фон негативный.
- **Чек из «Мой налог»** — единственная первичка для самозанятого, онлайн-касса не нужна.

**B2B SaaS.**
- Самозанятый **не подходит** (лимит 2.4 M ₽/год + запрет на долгую работу с юрлицами). Нужен ИП УСН или ООО.
- **152-ФЗ применим строго.** Дата рождения — ПДн. SaaS = обработчик + клиент-астролог = оператор. Серверы ПДн граждан РФ — в РФ.
- **Swiss Ephemeris** — dual license: AGPL (открытие всего кода SaaS) или коммерческая (~CHF 1550). Коммерческая лицензия экономически приемлема — окупается с первых подписчиков.
- **pyswisseph** под AGPL — при closed-source SaaS нужна коммерческая Swiss Ephemeris + проверка совместимости.

**Живому юристу/бухгалтеру:** текст оферты, политика возвратов, уведомление Роскомнадзора, оптимизация налог.режима, лицензионная схема SaaS.

**См. [B-legal.md](B-legal.md) для полного разбора.**

### C — Каналы

**Состояние каналов апрель 2026 (резко изменилось за зиму 2025-2026):**
- **Telegram** — ~100% заблокирован в РФ (Феб 2026+); 65 M россиян через VPN.
- **YouTube** — полностью заблокирован через DNS removal (Феб 2026). В РФ — через VPN.
- **Instagram** — заблокирован с 2022. Доступен за рубежом.
- **TikTok** — ByteDance отключил создание контента в РФ.
- **VPN** — 469 сервисов заблокировано, эскалация продолжается.
- **Работают без VPN:** Дзен (73 M MAU), VK (88-92 M), Rutube, MAX (100 M+), Boosty.

**Асимметрия внутри РФ vs диаспоры** — ключевое наблюдение.

| Канал | Клиент в РФ | Клиент в Германии/Японии |
|-------|-------------|---------------------------|
| Дзен | ✅ нативно | ⚠️ работает, не habitat |
| Telegram | ❌ VPN только | ✅ нативно |
| Instagram | ❌ VPN | ✅ нативно |
| Сайт | ✅ одинаково | ✅ одинаково |

**ВЫВОД.** **У Марины нет одного оптимального канала для обеих ЦА.** Стратегия: **двухканальная минимум**:
- Ядро РФ: Дзен + Дзен Премиум (уже работающий 5-летний актив).
- Ядро диаспоры: Telegram (+ Instagram опционально).
- **Сайт — единый single source of truth**, одинаково работает для обеих географий.

**B2B SaaS каналы:** партнёрства со школами (ключевое), реферальная программа, конференции, собственный Telegram/сайт для диаспоры астрологов.

### D — Сайт

**B2C.**
- Сайт — **не «визитка», а юридический + операционный актив + единая точка для всех каналов**.
- Минимальный MVP: offer, услуги с ценами, методика, trust-блок, отзывы, FAQ, форма записи + оплата, юр.блок.
- Стек: **Tilda** достаточно для лендинга с записью. WordPress / Taplink — альтернативы. Yclients не оптимален (избыточен для онлайн-услуги).
- **Mobile-first** — 70% Дзен с мобильных.
- **Предоплата 100%** — стандарт индустрии, убирает no-show.

**B2B SaaS.**
- Landing: features, pricing (референс Chronos 550/2000/10 001 ₽/мес), **публичное demo критично**, trial 7-14 дней.
- **Бренд-стратегия открытый вопрос:** один бренд Марины vs два бренда (как у Павла Андреева: LabLife + Chronos + личный бренд).

**Важно:** стек `Haskell core + Python services + React web` из `README.md` — **избыточен для B2C лендинга**, релевантен только для B2B SaaS (и даже там — требует обоснования).

### E — Оплаты

**B2C.**
- Самозанятый: СБП / карта / ЮKassa / Robokassa. Чек из «Мой налог» автоматически. Онлайн-касса не нужна.
- **Категория «астрология/эзотерика»** в платёжных агрегаторах имеет ограничения (нет рассрочек в ЮKassa/Prodamus). Базовый приём работает.
- **Международные клиенты (Германия, Япония):**
  - Прямые банковские переводы Европа→РФ практически **не работают** на апрель 2026.
  - Работающие варианты: crypto (USDT), платёжные агенты с транзитом, счета в нейтральных юрисдикциях.
  - Японии — практически только crypto.

**B2B SaaS.**
- Подписка (не pay-per-use) — доминирующая модель.
- Ценовая сетка ориентир: базовый 500-1000 ₽/мес, pro 2000-5000, enterprise 10 000+ — следует Chronos.
- Рекуррентные через ЮKassa работают, подключение вручную.
- Free tier + trial 7-14 дней — практически обязательны.
- Международные плательщики — Stripe недоступен, нужны Paddle / партнёр в ЕС / crypto.

**Money flow:** схема для обоих продуктов прописана в E-payments.md с документами.

**См. [E-payments.md](E-payments.md) для полного разбора.**

### F — Операционная модель

**B2C.**
- Воронка: контент → заявка → оплата → опросник (критично: точное время рождения) → созвон → запись + резюме.
- **Специфичная боль:** не все клиенты знают точное время рождения. UI должен роут-ить их в опциональную ректификацию (9-12 k ₽).
- **SLA:** ответ на заявку <24 ч, консультация 3-14 дней от оплаты, резюме 1-3 дня после сессии.
- Стек: Tilda + Calendly/Яндекс.Календарь + Google Forms + Zoom/Телемост + Google Drive + Notion/Sheets. Без собственной CRM на этой фазе.

**B2B SaaS.**
- Астролог как пользователь — НЕ типовой B2B. Длительные сессии (1-3 ч с картой), персонализация школы (Placidus/Koch/etc), заметки как IP.
- Минимальный MVP расчётов: натал, соляр, транзиты, синастрия, прогрессии/дирекции, дома.
- **Расчётные библиотеки:** Swiss Ephemeris (эталон, dual license), pyswisseph (AGPL), Skyfield (BSD, альтернатива без астро-абстракций).
- **Визуализация:** @astrodraw/astrochart — open-source TypeScript, готовое решение.
- **Стек:** Python (pyswisseph / Flatlib / Skyfield) — естественная экосистема. **Haskell** не имеет зрелых эфемерид — их разработка = 1-2 человеко-года. Архитекту — вопрос для Phase 1.

**См. [F-ops.md](F-ops.md) для полного разбора.**

### G — Стратегия

**Ключевое открытие:** паттерн «курсы + софт» **уже работает у основного конкурента** — LabLife (школа) + Chronos (софт) под Павлом Андреевым. Это не гипотетическая модель, а живой референс.

**Три работающих сценария:**

| Сценарий | Что | Когда подходит |
|----------|-----|----------------|
| A — только B2C | Консультации + курсы + Дзен, без SaaS | Если цель — стабильная выручка без риска актива |
| B — B2C сейчас → SaaS 6-12 мес | B2C закрепляет → SaaS параллельно с Мариной как первым пользователем | Баланс риска и амбиции |
| C — SaaS-first | Параллельная разработка SaaS с момента 0 | Если готовность к финансовому и техническому риску |

**Корневой критерий выбора:**
> Что важнее на горизонте 12-24 мес: (a) стабильная выручка без большого риска, или (b) построение масштабируемого актива с принятием риска?

Research **не выбирает** за клиента. Research описывает сценарии. Выбор — за клиентом + архитектором.

**Практическая последовательность (не зависит от сценария):**
1. Срочно (1-3 мес): сайт с приёмом + юр.гигиена.
2. 3-6 мес: Дзен Премиум / курс-эксперимент (проверить спрос на курсовой формат).
3. После 6 мес: решение о SaaS основывается на **собственной боли Марины** и **спросе из её аудитории**, не на гипотезе.

**Независимо от сценария:** курсы стоит планировать на Phase 1-2, не откладывать.

**См. [G-strategy.md](G-strategy.md) для полного разбора.**

## Для Марины лично (синтез)

1. **Сайт с приёмом оплаты и онлайн-записью** — must, пока нет. Это базовая упаковка любой дальнейшей деятельности.
2. **Не мигрировать с Дзена** — это её главный актив (5 лет, 5960 подписчиков, активный Премиум). Расширять, не переключаться.
3. **Telegram добавлять — но для диаспоры** (Германия/Япония — там он работает нативно; в РФ без VPN не пройдёт).
4. **Позиционирование продуктов:** «методика самопознания / интерпретация опыта», а не «предсказание». Защищает от юр.рисков одновременно.
5. **Триггер для запуска SaaS:** не гипотетический срок «6-12 мес», а комбинация **собственной боли от инструментов** + **запросы из её аудитории** «каким софтом пользуешься».
6. **Курсы — не отдельный продукт, а часть воронки**. Запускать раньше SaaS и использовать Дзен Премиум / GetCourse как платформу.

## Для architecture-фазы (открытые вопросы)

### Критические для архитектуры

1. **Стек для B2C MVP:** Tilda / WordPress / кастомный? (Не стек sitka.)
2. **Стек для SaaS (если Phase 2):** Python-first для расчётов — почти неизбежно. Haskell core — требует бизнес-обоснования. React — OK для web-фронта.
3. **Лицензия Swiss Ephemeris:** AGPL (open-source SaaS) vs коммерческая (CHF ~1550) vs Skyfield (BSD, но нужно строить астро-абстракции).
4. **Бренд-стратегия:** один сайт Марины или два сайта (личный + SaaS-бренд).
5. **Последовательность фаз:** A / B / C — до выбора сценария архитектура не должна over-engineer.

### Требующие бизнес-решения клиента

3. Риск-профиль (для выбора сценария).
