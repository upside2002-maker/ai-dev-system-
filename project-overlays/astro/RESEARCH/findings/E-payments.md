## E.1 — Оплата консультации (B2C)

### Базовый кейс: самозанятый + консультация физлицу

**ФАКТ.** Самозанятый принимает оплату:
- Банковским переводом на личную карту (счёт на имя физлица — не р/с юрлица).
- СБП-переводом (от клиента физлица).
- Через «Мой налог» — выписывает чек после поступления средств, сумма в рублях по курсу ЦБ (если валюта).

([secrets.tbank.ru](https://secrets.tbank.ru/voprosy-otvety/valyuta-samozanyatye/), [robokassa.com](https://robokassa.com/blog/articles/kak-prinimat-oplatu-s-inostrannykh-kart/), [e-kontur.ru](https://e-kontur.ru/enquiry/2088/samozanyatomu-rabotat-na-inostrannogo-zakazchika))

**ФАКТ.** Самозанятому **не нужна** онлайн-касса (54-ФЗ не применяется к НПД) — чек из «Мой налог» заменяет.

**ВЫВОД.** Для MVP Марины технически можно начать с **самого простого стека**:
- СБП-QR на сайте (генерируется через банк эксперта).
- Или ручной перевод по реквизитам карты.
- Чек — автоматически из «Мой налог» после поступления, отправляется клиенту.

Это 0 руб/мес подписки за платёжный агрегатор. Минус — низкая автоматизация (ручная сверка оплат).

### Платёжный агрегатор: ЮKassa / Robokassa / CloudPayments

**ФАКТ.** ЮKassa (Яндекс) поддерживает прием карт, СБП, рекуррентные платежи; подключение автоплатежей требует ручного запроса менеджеру ([yookassa.ru/regulyarnye-platezhi](https://yookassa.ru/regulyarnye-platezhi/)).

**ФАКТ.** Robokassa — альтернатива ЮKassa, работает с самозанятыми и ИП.

**ФАКТ (КРИТИЧНО).** У некоторых агрегаторов (Prodamus, банковские партнёры ЮKassa) **ограничен функционал для категорий «эзотерика, астрология, таро»**:
- Рассрочки не подключаются.
- Базовый приём платежей есть, но часть фичей (installment, BNPL) закрыта.

([vc.ru про Prodamus](https://vc.ru/services/2657267-prodamus-platezhnyj-modul-dlya-samozanyatyh-i-ip))

**ВЫВОД.**
- Приём платежей за астрологические консультации — **работает** через стандартные агрегаторы.
- При регистрации в ЮKassa / аналогах Марине нужно честно декларировать вид деятельности — это определит, какие функции включат.
- Рассрочек на консультации ждать не стоит; на курсы (education) — возможно (другая MCC-категория).

**ОТКРЫТЫЙ ВОПРОС (для живой проверки).** При регистрации в ЮKassa/Robokassa указывать «консультационные услуги» (96.09) или «астрология» (более узко) — от этого зависит степень ограничений. Задача для бухгалтера / менеджера платёжного агрегатора.

### Booking-сервисы со встроенной оплатой

**ФАКТ.** Yclients / YPLACES принимает оплату услуг онлайн ([play.google.com YPLACES](https://play.google.com/store/apps/details?id=com.yclients.yplaces&hl=en_US)). Интеграция стандартная для офлайн-индустрии.

**ФАКТ.** Lunaro принимает оплату своей платформой, забирает комиссию; астролог получает выплаты ([lunaro.ru](https://lunaro.ru/catalog/astrologi)).

**ВЫВОД.** Booking-платформы — путь, но они оттягивают на себя бренд и комиссии (10-30%). Для Марины с её собственным каналом это **не оптимальный путь**, разве что как дополнительный источник клиентов.

### Предоплата vs постоплата

**ФАКТ.** У практиков доминирует **100% предоплата** ([mylablife, astro-academia](https://mylablife.ru/consultations/)):
- Сначала оплата.
- Потом выдаётся опросник.
- Потом назначается слот.
- Потом — консультация.

**ВЫВОД.**
- Предоплата 100% — стандарт индустрии, **не спорный**.
- Снижает no-show почти до нуля — клиент, заплативший за слот, приходит.
- Исключение — проверенные постоянные клиенты, с которыми можно работать постоплатой в знак доверия (администрируется вручную Мариной).

### Возвраты

**См. блок B** — формулировка политики возвратов. Техническая сторона:
- Через ЮKassa/Robokassa — кнопка «возврат» в ЛК, деньги возвращаются на ту же карту за 3-10 дней.
- При переводе на карту самозанятого — обратный перевод вручную + аннулирование чека в «Мой налог».
- Необходимо фиксировать в оферте: сколько % удерживается при поздней отмене, срок возврата, на какую карту.

### Международные клиенты (Германия, Япония и др.)

**ФАКТ.** Самозанятый имеет право принимать оплату из-за границы ([secrets.tbank.ru](https://secrets.tbank.ru/voprosy-otvety/valyuta-samozanyatye/)):
- Через валютный счёт в российском банке.
- Через счёт в иностранном банке (с отчётностью в ФНС).
- Через SWIFT (долго, комиссионно).
- Чек в «Мой налог» — на рублёвый эквивалент по курсу ЦБ на день получения.

**ФАКТ.** С февраля-апреля 2026 прямой банковский перевод из Германии заблокирован у большинства Sparkasse / Commerzbank / Deutsche Bank ([vc.ru, 2026](https://vc.ru/money/2760853-kak-perevesti-dengi-iz-germanii-v-rossiyu), [dtf.ru, 2026](https://dtf.ru/howto/4844184-mezhdunarodnye-perevody-v-2026-godu)).

**ФАКТ.** Рабочие маршруты Германия → РФ:
- Через платёжного агента (Германия → ОАЭ/Армения/Казахстан/Турция → РФ) — 2-3 часа до суток.
- Через криптовалюту (легально-серая зона, требует осторожности; не для массового использования клиентами-обывателями).
- Прямой перевод в некоторые работающие банки (Газпромбанк имеет исключения по санкциям до конца периода, проверять актуальность).

**ФАКТ.** Япония — наиболее ограниченное направление. Работают только агентские маршруты через нейтральные юрисдикции ([dtf.ru](https://dtf.ru/howto/4844184-mezhdunarodnye-perevody-v-2026-godu)).

**ВЫВОД.** Для клиентов Марины в Германии / Японии / EU / других стран:

- **Не полагаться на банковские переводы из их страны напрямую в РФ** — это не работает или работает с огромными задержками.
- Рабочие варианты:
  1. **Криптовалюта / стейблкоины (USDT)**: клиент оплачивает в USDT, Марина получает на криптокошелёк, конвертирует в рубли через внутренний обменник. Легально при декларировании дохода. Работает мгновенно, ~1% комиссии. **Технически просто**, но клиенты-обыватели могут этого не уметь.
  2. **PayPal/Wise/Revolut** — в 2022-2024 обрезали операции с РФ, на апрель 2026 из РФ не работают для входящих. Не подходят.
  3. **Платёжный агент с транзитом** — сложно для разового клиента, удобно для регулярного.
  4. **Перевод на Марину через её иностранный счёт** (если есть) — если Марина имеет счёт в Германии / на Кипре / etc, клиент платит локально, Марина переносит себе в РФ отдельными операциями. Требует открытия зарубежного счёта — не всегда возможно.
  5. **Через другого посредника в нужной стране** — кто-то в Германии собирает оплаты локально и передаёт Марине через транзит.

**ВЫВОД (корневой принцип).** Идеального универсального решения для «клиент в Японии → рубли Марине» **нет** на апрель 2026. Это системный риск — не технический, а геополитический, Research его только фиксирует, не решает.

Практический ориентир:
- Для диаспоры Европа — агент Revolut→Казахстан→Тинькофф или crypto работают.
- Для Японии — crypto практически единственный массовый вариант.
- Для Марины: либо объявить фиксированный способ (например «USDT на такой-то адрес» + публичный курс конвертации), либо индивидуально договариваться с каждым клиентом.

**ОТКРЫТЫЙ ВОПРОС.** Нужно определить, какая доля клиентов Марины — диаспора, и какой объём они приносят. Если >20% выручки — стоит инвестировать в один из стабильных способов. Если <5% — индивидуальная договорённость.

### Что видит клиент при оплате

**ВЫВОД.** Для MVP сайта Марины достаточно:
- Кнопка «Записаться и оплатить» → страница ЮKassa/Robokassa → оплата по карте/СБП → возврат на сайт с подтверждением.
- Автоматическая отправка клиенту: чек, подтверждение сессии, ссылка на опросник с данными рождения, ссылка на онлайн-встречу.
- Вся последовательность занимает 1-2 минуты.

Для международных клиентов — отдельная страница с инструкцией («принимаем USDT на такой адрес, или связаться для индивидуального способа»).

## E.2 — Оплата SaaS (B2B, перспектива)

### Подписка vs pay-per-use

**ФАКТ.** Рынок показывает:
- **Chronos** — подписка (550/2000/10001 ₽/мес), без opt-on покупки отдельных расчётов.
- **Chani** — freemium + подписка.
- **Sanctuary** — подписка $19.99/$199.99.
- **AstroSeek** — freemium (бесплатно расчёты, опционально AstroPremium с доп.фичами).

**ВЫВОД.** Для B2B астрологов-практиков **подписка** — работающая модель. Pay-per-use — редкая и работает только для очень эпизодических пользователей (лучше маркетплейс типа Lunaro, не SaaS).

Ценовые тиры ориентиром:
- Базовый (~500-1000 ₽/мес) — для начинающих астрологов / студентов школы.
- Профессиональный (~2000-5000 ₽/мес) — для практикующих.
- Энтерпрайз (~10000+ ₽/мес) — для школ и профессиональных студий.

Этот ориентир повторяет Chronos — это не копирование, это свидетельство рабочей рыночной структуры.

### Рекуррентные платежи

**ФАКТ.** ЮKassa поддерживает рекуррентные платежи через карты и СБП. Для подключения — ручной запрос менеджеру ([yookassa.ru](https://yookassa.ru/regulyarnye-platezhi/), [u-kassa.online](https://u-kassa.online/)).

**ФАКТ.** Требования ЦБ РФ к рекуррентным платежам: согласие клиента в явном виде, возможность отмены в любой момент, уведомление о списании.

**ВЫВОД.** Технически реализуемо. Administrative complexity средняя — подключение вручную. Для B2B SaaS — обязательно (ручной счёт каждый месяц клиенту — убийственно для retention).

### Международные подписки на SaaS (если клиенты есть)

**ФАКТ.** Stripe недоступен российским юрлицам с 2022.

**ФАКТ.** Альтернативы для приёма международных платежей на SaaS-сервис в РФ:
- Партнёр-эквайер в UAE / Казахстане / Армении с холдингом или платёжным агентом.
- Crypto (USDT, USDC) как опция — для нишевых клиентов технически приемлемо.
- Paddle / Lemon Squeezy — работают как MoR (merchant of record), но принимают ли плательщиков в Россию — проверить (обычно сервис для западных SaaS).

**ВЫВОД.** Для международной B2B-продажи нужна зарубежная юрисдикция партнёра или crypto. В рамках MVP B2B SaaS — **не гнать** в международную экспансию до стабилизации РФ-выручки. Фокус на русскоязычных клиентах в РФ (они платят картой / СБП) + в диаспоре (они платят из-за границы через crypto / иностранные карты через Paddle / личный счёт SaaS в ЕС).

### Trial / Free tier

**ФАКТ.** Chronos даёт неделю бесплатно при первой подписке ([chronos.mg](https://chronos.mg/)).

**ФАКТ.** Astro.com, AstroSeek — freemium (большая часть функций бесплатно).

**ВЫВОД.** Для SaaS Марины:
- **Free tier** (бесплатные базовые расчёты натальной карты без сохранения) — рекомендуется. Это одновременно lead-magnet и снижение барьера первого пробования.
- **Triale** (7-14 дней полного доступа без оплаты) — обязателен для продажи.
- **Жёсткая стена «оплатите чтобы увидеть хоть что-то»** — проиграет Chronos в этой нише.

### Политика возврата SaaS

**ВЫВОД.** Типичная модель: pro-rated refund при отмене в середине периода (вернуть за оставшиеся дни). Не обязателен — можно политикой «оплата невозвратна, но подписку можно отменить до следующего периода». Рынок SaaS оба варианта принимает.

## E.3 — Money flow

### B2C (консультация)

```
Клиент (Дзен/Telegram/Instagram)
  ↓
 Сайт Марины (заявка + выбор слота)
  ↓
 ЮKassa/Robokassa (оплата карта/СБП)
        ИЛИ USDT-кошелёк (для диаспоры)
  ↓
 Подтверждение оплаты → автоматические действия:
  - Отправка опросника по данным рождения (на email)
  - Выдача ссылки на созвон (Zoom/Google Meet/Телемост)
  - Создание записи в календаре Марины
  - Выписка чека из "Мой налог" → на email клиенту
  ↓
 Консультация (созвон 1-2 часа) с записью
  ↓
 Post-session: письменное резюме + аудиофайл клиенту
  ↓
 (опционально) Follow-up через 1-2 мес — доп.консультация / апсел
```

**Документы:**
1. Оферта (публичная на сайте, считается акцептом при оплате).
2. Согласие на обработку ПДн (checkbox при заявке).
3. Чек из «Мой налог» (автоматически после оплаты).
4. Запись консультации (храниться у Марины, ссылка клиенту).
5. Письменное резюме (в email клиенту).

### B2B SaaS (перспектива)

```
Астролог (клиент SaaS)
  ↓
 Landing → Sign up (email + пароль)
  ↓
 Free tier использование / Trial 7 дней
  ↓
 Подписка выбрана → ЮKassa рекуррентный платёж
  ↓
 Ежемесячное списание → чек / УПД на email
  ↓
 Использование сервиса (расчёты, сохранение карт клиентов)
  ↓
 Отмена: ЛК кнопка "Отменить подписку" → работает до конца периода
```

**Документы:**
1. Лицензионный договор / договор оказания услуг (публичная оферта).
2. Политика обработки ПДн (обязательна при регистрации клиентов).
3. Договор-приложение для SaaS ↔ клиент (в нём: роли оператор/обработчик по 152-ФЗ, условия хранения ПДн конечных клиентов астролога).
4. Ежемесячный чек / УПД.

## Перекрёстная проверка с блоком B

**Проверка 1:** самозанятый → максимум 2.4 M ₽/год. При чеке 10 000 ₽ — это 240 консультаций/год (~20/мес). Предел достижим у активного практика. Для SaaS с множеством подписчиков — самозанятость не подходит.

**Проверка 2:** рассрочки по астрологии не везде включают → курс 100 000 ₽ без рассрочки может отбить часть спроса. Решение — самостоятельная pay-in-2 схема: 50% предоплата, 50% к середине курса.

**Проверка 3:** международные клиенты → для самозанятого легально через Мой налог, но физически приём валюты усложнён. Не противоречит B, подтверждает: для устойчивого международного потока нужен Boosty / платёжный агент / crypto.

**Проверка 4:** SaaS с рекуррентом требует ИП/ООО (согласуется с B.2: режим НПД не годится для постоянных юрлиц-клиентов и рекуррента).
