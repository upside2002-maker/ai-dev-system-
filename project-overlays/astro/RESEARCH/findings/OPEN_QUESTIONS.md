# OPEN_QUESTIONS — что требует живого специалиста

**Дата:** 2026-04-17.

AI-ассистент сформулировал вопросы и варианты. **Финальные решения не могут быть приняты без профильных специалистов**. Это прямое применение принципа CLAUDE_GLOBAL: «AI не заменяет живого юриста / бухгалтера / отраслевого консультанта».

Вопросы сгруппированы по специалистам.

---

## Живой юрист (приоритет P0)

### Юрист по потребительскому праву + IT-право

1. **Текст договора-оферты на астрологическую консультацию.**
   - Формулировка предмета услуги («интерпретация натальной карты методом западной классической астрологии», **НЕ** «предсказание»).
   - Политика возвратов при отмене клиентом / переносе / no-show.
   - Разграничение ответственности: эксперт не несёт ответственности за решения клиента, принятые на основе интерпретации.
   - Блок про ПДн (согласие на обработку).
   - Типовой срок оферты, юрисдикция, порядок разрешения споров.

2. **Политика обработки персональных данных.**
   - Для B2C-сайта Марины (собирает ФИО, даты рождения, места, email, иногда запросы клиентов).
   - Для B2B SaaS (хранит данные клиентов астрологов-пользователей).
   - Уведомление Роскомнадзора: процедура, сроки, форма.

3. **Роли оператор / обработчик в SaaS-контракте.**
   - SaaS как обработчик по поручению астролога-оператора.
   - Требование хранения ПДн граждан РФ на серверах в РФ (ст. 18 ч. 5 152-ФЗ) + конфликт с клиентами-астрологами из диаспоры, обрабатывающими данные РФ-клиентов — нужна юр.развязка.

4. **Выбор юридической конструкции для SaaS с курсами.**
   - Один ИП/ООО на «SaaS + курсы» или раздельные юрлица?
   - Лицензионный договор на ПО vs договор оказания услуг — плюсы/минусы для астрологического SaaS.
   - Нужна ли образовательная лицензия для курсов (если выдаются сертификаты Минобрнауки-стандарта) vs неформальные курсы без дипломов.

5. **Анализ публичных оферт действующих астрологов-ИП** — нужен ли прецедентный анализ (Мария Штерн, Лиза Васина, Павел Андреев, Lunaro) для калибровки текста Марины.

6. **Законопроект № 901048-8 о запрете рекламы эзотерики** — мониторинг статуса, сценарии при принятии. Какие практические защитные формулировки в бренде/рекламе Марины сейчас снижают риск потерь при возможном принятии.

7. **Swiss Ephemeris лицензирование.**
   - Актуальные условия коммерческой лицензии Astrodienst (CHF 750-1550 по состоянию август 2022 — возможно обновилось).
   - Совместимость pyswisseph (AGPL) с коммерческим licensee Swiss Ephemeris — неразрешённый open issue на GitHub ([pyswisseph #92](https://github.com/astrorigin/pyswisseph/issues/92)), требует прямого контакта с Astrodienst и pyswisseph-мейнтейнерами.
   - Альтернатива — переписать расчётное ядро под Skyfield (BSD), потеря совместимости с классическими реализациями.

## Бухгалтер (приоритет P0)

1. **Оптимальный налог.режим для Марины на старте B2C.**
   - Самозанятый (НПД) vs ИП УСН 6% — с учётом международных клиентов, прогноза выручки, планов на курсы.
   - Критерии триггера перехода на ИП (конкретные финансовые метрики).

2. **ОКВЭД для ИП в будущем.**
   - 96.09 основной (астрологические услуги).
   - При запуске курсов — 85.41.9 / 85.42 или иной?
   - При SaaS — 62.01 «Разработка ПО»?

3. **Валютный учёт.**
   - Приём платежей от клиентов из Германии/Японии для самозанятого — процедура (чек в «Мой налог», курс ЦБ, документы-основание).
   - Лимит ЦБ РФ $1M/мес + триггеры запроса банка о происхождении средств.

4. **Льготы по НДС / налог на прибыль для SaaS.**
   - Актуальные условия 2026 для включения в реестр российского ПО.
   - Режим АУСН — подходит ли.

5. **Дивиденды vs зарплата vs вывод.**
   - Если Марина перейдёт на ИП УСН — как оптимально выводить средства.

## Астролог-практик (приоритет P1)

Не для окончательного решения юр.вопросов, а для **валидации предположений** о поведении ЦА и практиков:

1. **Спрос на SaaS у практикующих астрологов в РФ.**
   - Кто сейчас использует Chronos / ZET / Астронова / Astro.com / собственные решения?
   - Какие типовые жалобы на эти инструменты?
   - Что заставило бы сменить основной инструмент?

2. **Курсы как канал.**
   - Формат платных материалов у коллег: видеоуроки / текст / вебинары / смешанный?
   - Ценовой диапазон курсов для начинающих / продвинутых.
   - Какая ожидаемая конверсия из Дзен-подписчика в клиента курса?

3. **Форматы консультаций.**
   - Длительность типовой консультации у практиков (1 час / 1.5 / 2+).
   - Соотношение разных видов (натал vs соляр vs синастрия vs прогностика).
   - Что делают с клиентом, который не знает точного времени рождения (отказ / ректификация / приблизительная работа)?

4. **Постконсультационные материалы.**
   - Что типично даётся клиенту (запись / письменное резюме / PDF-отчёт / дневник транзитов)?
   - Формат follow-up.

5. **Диаспора как ЦА.**
   - Опыт работы с русскоязычными клиентами в Европе/Японии/США — отличия от РФ-клиентов.
   - Запросы, которые специфичны для диаспоры (эмиграция, адаптация, возвращение).

**Формат сбора:** 5-10 полуструктурированных интервью с практикующими астрологами. Марина может опросить коллег из своего круга / Дзен-аудитории.

## Маркетолог-технолог (приоритет P2)

1. **Benchmark конверсий на Дзене.**
   - Типовая конверсия Дзен-подписчика → клиента экспертной услуги. Не нашёл открытых данных.
   - Время удержания внимания в Дзен Премиум (месячный churn).

2. **Эффективность каналов диаспоры.**
   - Telegram в Германии/Японии — какие форматы контента у русскоязычных блогеров-экспертов работают?
   - Адаптация Дзен-контента для Telegram-канала — избегать простого дубляжа.

3. **Дзен Premium vs Boosty для Марины.**
   - Какая платформа даёт большую LTV в эзотерическом сегменте.
   - Возможность параллельного использования (контент для одной, закрытый контент для другой).

## Специалист по ПДн / 152-ФЗ (приоритет P1)

Если запускается SaaS:

1. **Юрисдикционная развязка для клиентов SaaS из диаспоры.**
   - Астролог в Берлине обрабатывает ПДн россиянина — где должны храниться данные? РФ (152-ФЗ применим) или ЕС (GDPR применим) или дуальное хранение?
   - Это open question, который не решается одним юристом — нужен специалист по cross-border data protection.

2. **Договорная структура subprocessor-relationship** (если SaaS использует зарубежные сервисы: хостинг, аналитика, email).

3. **Уведомление Роскомнадзора** — форма при работе как обработчик ПДн для множества клиентов-операторов.

## Технический партнёр / CTO (приоритет P1 для сценариев B/C)

1. **Выбор эфемерид:** Swiss Ephemeris AGPL vs коммерческая vs Skyfield — технико-экономическое сравнение.

2. **Стек backend:** Python-first для расчётов (почти неизбежно) vs попытка Haskell core (экономическое обоснование под сомнением).

3. **Архитектура MVP SaaS:** monolith vs начальное разделение services. Кажется, начинать с monolith (PostgreSQL + FastAPI + React), модуляризация при необходимости.

4. **Offline-поддержка:** PWA достаточна или native app нужно? Зависит от поведения целевых астрологов.

5. **LMS для курсов:** собственная vs GetCourse / Antitreningi / продвинутый Notion + Stripe.

## Бизнес-данные клиента (Марина — внутренняя информация)

1. **География текущей клиентской базы** — какой % в РФ, Германии, Японии, других?
2. **Средний чек и кол-во консультаций в месяц сейчас** — для калибровки экономической модели.
3. **Наличие действующих курсов / тестов курсового формата** — если запускала — что получилось.
4. **Финансовые цели** — сумма выручки через 12 мес, через 3 года.
5. **Готовность инвестировать** собственные средства в SaaS разработку — ориентир (200 k / 1 M / 3 M+ ₽).
6. **Наличие технического партнёра** — есть ли кто-то в окружении, кто готов взять техническую часть.
7. **Отношение к конкуренции с Chronos/LabLife** — вступать ли в прямую конкуренцию или искать незанятую поднишу.

## Что **не** требует живого специалиста (в этом research закрыто)

- Существование рынка — подтверждено.
- Технические варианты расчётных библиотек — описаны с лицензиями.
- Состояние каналов на апрель 2026 — зафиксировано с источниками.
- Базовая гигиена платежей — описана с предупреждениями по категории «эзотерика».

## Приоритизация

**Критический путь P0** (без этого Phase 1 не стартует):
- Юрист по потребительскому праву: оферта + политика возвратов.
- Бухгалтер: налог.режим + ОКВЭД.

**Желательно P1** (можно запустить параллельно с архитектурным проектированием):
- Астролог-практик: 5-10 интервью.
- Специалист по ПДн.
- Технический партнёр / CTO.

**Менее срочно P2** (можно отложить до первых alpha-клиентов SaaS):
- Маркетолог-технолог.

## Формат «вопросов для специалиста»

При обращении к юристу/бухгалтеру — **не** задавать открытые вопросы типа «что нам делать?». Задавать конкретные:

✅ «Вот текст черновика оферты — проверьте риски и предложите корректировки».
✅ «Вот список услуг и прогноз выручки — какой налог.режим оптимален на горизонте 1 года, 3 лет?».
✅ «Мы планируем принимать оплату из Германии — опишите легальные маршруты для самозанятого».

❌ «Как нам запустить бизнес?»
❌ «Что выбрать — самозанятость или ИП?»

AI-research подготовил контекст, чтобы специалист работал не с чистого листа, а с уже сформулированными вариантами.
