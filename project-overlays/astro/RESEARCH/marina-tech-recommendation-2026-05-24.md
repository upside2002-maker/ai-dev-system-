## 1. Frontend-стек: Astro + Cloudflare Pages — ПОДТВЕРЖДАЮ

### Плюсы Astro

- **Идеально для контентного сайта.** Astro заточен под static-content с фрагментами интерактивности (islands architecture).
- **Markdown-friendly.** Посты пишутся как `.md` или `.mdx` файлы — идеально для Марины (потом — для CMS-обвязки).
- **Mobile-first из коробки.** Минимум JS = быстрая загрузка на смартфоне (75% её аудитории).
- **TypeScript support** — для Worker'а это знакомая среда.
- **Хороший SEO** — статическая генерация, идеальные meta-теги, sitemap, RSS.

### Плюсы Cloudflare Pages

- **Бесплатно.** Free tier: unlimited bandwidth, 500 builds/month, custom domain, auto-SSL.
- **CDN из коробки** — мгновенная загрузка по всему миру.
- **Git-based deploy.** Push в репозиторий → автоматический деплой через 30-60 секунд.
- **Pages Functions** (free tier) — для форм захвата и проксирования API.
- **R2 Storage** (free до 10 ГБ) — если понадобится хранить PDF lead magnets.

### Альтернативы, которые рассмотрел и отверг

| Альтернатива | Почему не подходит |
|---|---|
| **Next.js + Vercel** | Тоже бесплатный free tier, но Next.js — overkill для контент-сайта без SSR. Vercel бесплатный, но строже лимиты по bandwidth (100 ГБ/мес против ∞ у CF). Astro проще. |
| **WordPress + хостинг** | Хостинг = деньги (~300-1000 ₽/мес минимум), безопасность требует обслуживания, скорость хуже. Бюджет 0 = не вариант. |
| **Tilda / Webflow** | Платные платформы, $14-$36/мес. Бюджет 0 = не вариант. Плюс Марина уже отказалась от Tilda. |
| **Чистый HTML + GitHub Pages** | Минимально, но нет CMS — каждый пост придётся писать вручную. Тяжело масштабироваться. |
| **Hugo, Eleventy, Jekyll** | Альтернативные SSG. Astro выигрывает по DX, TS-поддержке и community 2026 года. |

**Решение:** Astro + Cloudflare Pages. **Подтверждаю выбор Admin/Owner.**

## 2. CMS-механизм для Марины — ГЛАВНЫЙ выбор

Марина не-разработчик. Она не должна редактировать YAML frontmatter руками или собирать сайт командной строкой. Нужен слой между ней и git.

### Кандидат №1 — Decap CMS (бывший Netlify CMS)

**Что это:** бесплатный open-source headless CMS, который ставится как часть сайта (плагин Astro). Марина заходит на `marina-astrolog.pages.dev/admin`, видит WordPress-подобный интерфейс, пишет/правит посты в WYSIWYG-редакторе, нажимает «Опубликовать» — Decap делает git-коммит, CF Pages перестраивает сайт за 30-60 секунд.

**Плюсы:**
- Бесплатно навсегда.
- Знакомый Марине интерфейс админки (поля, превью, кнопки).
- Данные хранятся как `.md` в git — полный контроль, можно вернуть любую версию.
- Поддерживает WYSIWYG, картинки, превью.
- Большая community, проверен временем.

**Минусы:**
- Требует настройки OAuth (для редактирования git нужна авторизация). Для CF Pages — нужен либо OAuth-bridge на CF Workers (бесплатно, но нужно настроить), либо GitLab/GitHub OAuth напрямую.
- UI стареющий (выглядит как 2019 год).
- Нужен интернет + браузер для редактирования (не offline).

### Кандидат №2 — GitHub web-editor + GitHub Mobile

**Что это:** Марина редактирует .md файлы прямо в браузере на github.com, или через приложение GitHub Mobile. Каждое сохранение = коммит.

**Плюсы:**
- Ноль настроек, ноль зависимостей.
- Полностью бесплатно.
- Работает на телефоне через GitHub Mobile.

**Минусы:**
- Марине нужен GitHub-аккаунт (создаётся за 2 минуты).
- Нужно объяснить интерфейс GitHub (она увидит «commit», «pull request» — может смутить).
- Нет визуального превью — только сырой markdown.
- При ошибке в YAML frontmatter (`---` блок в начале файла) — пост не соберётся, нужно вернуться и поправить.

### Кандидат №3 — TinaCMS

**Что это:** git-based CMS с inline-редактором. Похож на Decap, но более современный UI и хуже OSS-история (часть фич платная).

**Плюсы:**
- Современный UI, inline-редактирование «на сайте».
- Хорошая интеграция с Astro.

**Минусы:**
- Free tier ограничен (2 пользователя, базовые фичи).
- Cloud-зависимый — данные через их API.
- Меньше community.

### Кандидат №4 — Sanity Free / Strapi

**Что это:** «настоящий» headless CMS как сервис.

**Плюсы:**
- Самый удобный UI.
- Богатые фичи (картинки, превью, типы контента).

**Минусы:**
- Данные не в git — на их сервере. При оттоке от сервиса — миграция боль.
- Free tier ограничен.
- Платное за рост.
- Излишне для текущего объёма (24 поста / Q3).

### Рекомендация BA

**Decap CMS** — основной выбор. Логика:
1. Бесплатно навсегда.
2. Данные в git = контроль и переносимость.
3. UI похож на админку, Марина не пугается.
4. Worker настраивает один раз, дальше Марина работает сама.

**Запасной путь** на случай если Decap не запустится за разумное время в спринте 1: **GitHub web-editor**. Марина с инструкцией на 1 страницу справится. Decap всегда можно докрутить во 2-м спринте.

## 3. Email-сервис для welcome-серии и рассылки

### Кандидат №1 — MailerLite

**Free tier:** 1 000 подписчиков, 12 000 писем/месяц, drag-and-drop редактор, автоматизация (welcome-серии).

**Плюсы:**
- Очень щедрый free tier.
- Простая интеграция через embed-форму.
- Автоматизации работают на free.

**Минусы:**
- Маркетинговые шаблоны простоватые (но это и плюс — быстрее настроить).
- Платный после 1 000 подписчиков (~$10-30/мес).

### Кандидат №2 — Brevo (бывший Sendinblue)

**Free tier:** unlimited подписчиков, 300 писем/день (= ~9 000/мес).

**Плюсы:**
- Безлимит подписчиков на free.
- Можно купить SMTP-кредиты дополнительно.

**Минусы:**
- 300 писем/день = жёсткий cap. При резком росте (рассылка по 1000 подписчиков) — нужно растянуть на 3-4 дня или платить.

### Кандидат №3 — Resend + кастомные шаблоны

**Что это:** transactional email (3 000 писем/мес free), используется через API из CF Workers.

**Плюсы:**
- Идеально для подтверждений (запись на консультацию, скачивание lead magnet).
- Программируемый.

**Минусы:**
- Нужен код, нет визуального редактора.
- Welcome-серии писать вручную.

### Рекомендация BA

**MailerLite на старте.** Логика:
1. 1 000 подписчиков — комфортный запас (Марина при 5968 ТГ-подписчиков сейчас ловит ~50-100 email-подписок в месяц после запуска воронки).
2. Готовые welcome-автоматизации.
3. Embed-форма прямо вставляется в Astro.
4. При росте до 800-900 подписчиков — за месяц переходим на платный тариф ($10-30/мес, минимальная инвестиция к тому времени).

Параллельно — **Resend для транзакционных писем** (подтверждение записи на консультацию, скачивание PDF). Это разделение труда: marketing emails — MailerLite, system emails — Resend.

## 4. Аналитика

**Яндекс Метрика** — рекомендация. Логика:
- Уже стоит на Дзен-канале Марины (счётчик 92473535). Она знакома с интерфейсом.
- В её аккаунте уже 3 счётчика — добавить четвёртый (на новый сайт) = два клика.
- Бесплатно, неограниченно.
- Поддерживает все нужные цели (формы, клики, скачивания).

**Дополнительно — Google Analytics 4** (опционально):
- Если у Марины есть Google-аккаунт — добавить как параллельный.
- Дублирует данные, полезно для долгосрочной аналитики и комфорта если она уйдёт от Яндекса.

**Не нужно (пока):** Plausible / Fathom / Posthog — они классные, но Метрики достаточно для текущих задач, и они бесплатные.

## 5. Платёжная система

### Краткосрочно (MVP, первые 1-2 месяца)

**Ручной перевод** через карту или СБП-ссылку:
- Минимум настроек.
- Марина получает оплату на свою карту/счёт.
- Минус — не масштабируется, требует ручной обработки каждой записи.

### Средне-долгосрочно (после 5-10 консультаций/месяц)

**ЮKassa или CloudPayments через ИП / самозанятого:**
- Комиссия ~3% (ЮKassa) / 2.6% (CloudPayments).
- Требует Марине оформить ИП или самозанятого (1-2 дня в России, бесплатно).
- Автоматическое подтверждение оплаты → автоматическое письмо подтверждения через Resend → запись в CRM.

**ЮKassa или CloudPayments — почти эквиваленты.** ЮKassa чуть более популярна, чуть дороже. CloudPayments чуть дешевле, чуть меньше документации.

**Рекомендация BA:** **ЮKassa**, потому что:
- Большее community и больше тех, кто помог с настройкой.
- Поддержка в РФ из коробки.
- Прозрачная документация.

Альтернатива — Tinkoff Эквайринг (если у Марины уже ИП в Тинькофф).

## 6. Хранение PDF lead magnets

Варианты:
1. **Прямо в Astro репозитории** в папке `/public/lead-magnets/` — простейший путь. PDF ~500 КБ × 4 = 2 МБ. Норм.
2. **Cloudflare R2** — бесплатно до 10 ГБ, удобно для аналитики скачиваний.
3. **Google Drive публичная ссылка** — нет учёта скачиваний, не вписывается в воронку.

**Рекомендация:** на старте — в `/public/lead-magnets/`. Когда понадобится учёт скачиваний (статистика на каждый PDF) — мигрировать в R2.

## 7. Форма записи на консультацию

**Подход:** Astro-страница `/booking` с обычной HTML-формой, отправка через CF Pages Function на email Марины (через Resend) + одновременно на webhook MailerLite (для сегментирования).

**Альтернатива:** интеграция Tally / Typeform (free tier). Минус — отвлекает с сайта, плюс — готовый UI.

**Рекомендация:** кастомная форма в Astro (5 строк кода, полный контроль, mobile-first, без редиректов).

## 8. Календарь записи (для выбора даты консультации)

Опционально для MVP, обязательно через 2-3 месяца.

**Кандидаты:**
- **Cal.com** — open source, free self-hosted или free 1 user tier.
- **Calendly** — free tier (1 calendar type).
- **YClients** — российский сервис, free до X записей.

**Рекомендация:** на старте — без календаря, Марина пишет вручную с предложением 2-3 слотов. После 10-15 консультаций/месяц — Cal.com (бесплатно, под её домен).

## 9. Резюме — итоговый стек

| Слой | Решение | Free tier | Когда нужны деньги |
|---|---|---|---|
| Frontend | **Astro framework** | ∞ | никогда |
| Хостинг + CDN | **Cloudflare Pages** | ∞ bandwidth | при невероятном росте |
| Домен | `marina-astrolog.pages.dev` бесплатно ИЛИ `marina-arhipova.ru` за 500₽/год | — | сразу за домен |
| CMS | **Decap CMS** (с GitHub web-editor запасной) | ∞ | никогда |
| Email маркетинг | **MailerLite** | 1 000 подписчиков | ~$10-30/мес при 1000+ |
| Email транзакции | **Resend** | 3 000 писем/мес | $20/мес при росте |
| Аналитика | **Яндекс Метрика** | ∞ | никогда |
| Платежи | Ручной перевод → **ЮKassa** через ИП | — | комиссия ~3% после регистрации |
| PDF lead magnets | `/public/lead-magnets/` в репо | ∞ | никогда |
| Календарь записи | Без на старте → **Cal.com** | ∞ | при росте |

**Итого фиксированных трат для запуска:** 0 ₽ (если бесплатный поддомен) или 500 ₽/год (если свой домен).

**Текущие комиссии после раскрутки:** ~3% с каждого платежа через ЮKassa.
