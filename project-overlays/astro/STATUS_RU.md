## На очереди

Зависит от ответа Марины. Возможные направления:

- Если попросит ещё правки PDF (текст / разметка / визуал) — оформляем через задачу, классификация по риску: правки разметки и текста — класс C, правки числовых расчётов или Haskell-ядра — класс A.
- Если PDF её устраивает — три кандидата из бэклога:
  - **`solar-nodes-lilith-retro-display`** — класс A, расширение Planet ADT в Haskell-ядре + cascade по bright-line #8 (контракты JSON-схем + fixtures + roundtrip-тесты). Глифы уже подготовлены в `wheel_glyphs.py` во время итерации 1 wheel swap.
  - **`consultation-summary-matrix-rewrite`** — переработка матрицы повтора домов по методам.
  - **`section-order-recon`** — page-by-page diff с референсом Марины для финальной структурной сверки.

Приоритет между ними — определит Марина после просмотра.

## Не делаем сейчас

- **«Solar nodes / Lilith / retro display» без явного запроса от Марины.** Это серьёзная задача класса A (Haskell-ядро + schema cascade), не «давайте сделаем потому что глифы уже есть». Триггер запуска — её просьба видеть эти точки в карте, не наша инициатива.
- **Публичный хостинг репозитория** (GitHub / GitLab / Gitea). Без твоего отдельного «ок» не добавляем. Сейчас локальной копии и резервной копии (`/Users/ilya/Backups/astro.git`) достаточно.
- **SaaS-направление и B2C-сайт.** Сняты после исследования и разговора с Мариной — проект зафиксирован как внутренний инструмент. Возвращаться — отдельное продуктовое решение, не молчаливый дрейф.
- **Полная сверка архитектурного документа с фактическим кодом.** Откладывается до явного «давай разберёмся». Не блокирует.
