## 1. Резюме

Новый раздел консультации: по натальной карте выдаём **2–3 лучших сочетания «планета + дом (+ знак)»** для профессии + черновое толкование. Метод Дарагана: 5 стадий вокруг 4 «денежных» домов (X/II/VI/VIII). **Вся математика и отбор — в ядре** (новый `Domain.Vocation`), **толкование — редактируемые правила-данные** (правит Марина), **выдача — раздел консультации** (PDF + UI) по образцу возвратов. Делаем **по методу, не по мотивам** — якорим на страницы книги + golden на реальных картах.

## 4. Целевая архитектура (границы строго)

### 4.1 Core (Haskell) — вся математика и отбор
Новый `Domain.Vocation` (+ под-модули по необходимости: `Domain.Vocation.Satellites`, `Domain.Vocation.JonesPattern`, `Domain.Vocation.FixedStars`):
- **Стадия 1 (реализация):** для домов X/II/VI/VIII собрать признаки (а) управитель, (б) планеты в доме, (в) соединение/гармония с управителем, (г) взаимная рецепция.
- **Стадия 2 (способности):** сильные планеты (Asc-управитель + I дом; спутники Солнца; обитель/экзальтация; управитель солнечного знака; соединение/гармония с Солнцем; элевация; фигура Джонса; благие фикс.звёзды). Счётчик повторов (планета в нескольких списках = значимее).
- **Стадия 3 (фильтрация):** пересечение «способности ∩ реализация»; иерархия домов угловой(X)>последующий(II)>падающий(VI); VIII только при очень удачных.
- **Стадия 4 (выбор):** ранжирование → топ-2–3 «планета+дом», приоритет связи с X.
Чисто, без I/O. Принимает уже-резолвленные натальные позиции + куспиды (как `runSolar`/`runReturns`). **AST-ARCH-1: значимость-логику в Python НЕ плодим** — всё здесь.

### 4.2 Контракт (greenfield)
Новый `packages/contracts/vocation-*.schema.json` (вход = натал + куспиды; выход = таблица стадий 1–2 + топ-сочетания с факторами + ссылки на правило толкования). JD/числа, не текст. По образцу `returns-*`.

### 4.3 Толкование — правила-данные (правит Марина)
Стадия 5 — НЕ код. `packages/rulesets/vocation/*.{json,yaml}` (или поле в схеме): планета→сфера (Марс→спорт/борьба, Венера→искусство/сцена…), дом→способ дохода, знак→сужение (Марс+вода→водный спорт/море). Базовый черновой набор помечен «правится Мариной». Движок выдаёт сочетание + ключи; текст подставляется из правил.

### 4.4 Выдача — раздел консультации (PDF + UI)
Reuse пайплайна возвратов: Python собирает вход → один вызов ядра (`workflow:"vocation"` через Main dispatch) → раздел в `solar.html.j2` + UI-панель. JD→текст и подстановка правил — презентация.

## 5. Фазы (по образцу возвратов)

### Phase 0 — Stage 0 proof (НЕ продуктовый код)
Worker читает книгу (282–300, 206–212, 93–97, 69–79). На реальной карте (Марина person 4 + ещё 1–2) **вручную** прогоняет 5 стадий, строит таблицу 4×4, выводит топ-2–3. Сверяет с worked-example книги (если есть) и/или с тем, что Марина считает верным. **STOP-гейт:** если метод не воспроизводится / факторы неоднозначны / нет эталона для проверки «по методу» — STOP, эскалация (нужны reference-разборы Марины). Документирует точное reuse-vs-build по каждому фактору + конвенции (рецепция, спутники Солнца, фигуры Джонса, фикс.звёзды) со ссылками на страницы.

### Phase A — ядро (Tier A strict + независимый Reviewer)
`Domain.Vocation` (+ под-модули) стадии 1–4 + unit/golden-тесты на 2–3 реальных картах (математика воспроизводима, факторы заякорены). Достроить: спутники Солнца, фигуры Джонса, благие фикс.звёзды, рецепция (если нет), таблица+ранжирование. Schema-gate (vocation-контракт + Bridge.Vocation + Main dispatch + golden + TS) — атомарно.

### Phase B — толкование + выдача (Tier B)
Стадия 5 как правила-данные (черновой набор, «правится Мариной») + раздел консультации (PDF + UI). Reuse возвратов.

### (Следующий трек, НЕ в эпике) — полное толкование в голос Марины.

## 6. Риски

| Риск | Снижение |
|---|---|
| **Метод «по мотивам» вместо «по методу»** | Phase 0 читает книгу + ссылки на страницы в HANDOFF; golden на реальных картах; расхождение с книгой → верить книге |
| **Нет эталона для проверки корректности отбора** | Phase 0 STOP-гейт: нужен worked-example книги ИЛИ reference-разбор Марины. Иначе тестируем только воспроизводимость, не правильность |
| **Фигуры Джонса субъективны** | реализовать строго определённые случаи (bucket/handle, sling) по книге; пограничные — флаг + сверка |
| **Благие фикс.звёзды — какой список** | взять из книги Дарагана, не выдумывать |
| **VIII дом** | по методу — только при очень удачных сочетаниях; зашить как низкий приоритет/гейт |
| **Призвание навязывается** | не выдавать «призвание» где его в карте нет (DoD) |

## 8. Оценка

- Phase 0 (proof): малая, но критичная (метод-фиделити гейт).
- Phase A (ядро): **крупная** — несколько новых факторов (спутники, Джонс, фикс.звёзды, рецепция) + таблица/ранжирование + schema-gate. Tier A.

- Итого: Tier A эпик, 3 фазы, независимый Reviewer на Phase A.
