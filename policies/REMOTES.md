# Удалённые копии репозитория

У `ai-dev-system` две удалённые копии. Каждая решает свою задачу.

## origin — приватный репозиторий на GitHub

Адрес: `https://github.com/upside2002-maker/ai-dev-system-.git` (имя с дефисом в конце — так создан владельцем).

Видимость: приватный. Доступ есть у владельца (`upside2002@gmail.com`) и у второго разработчика (`bondvit@gmail.com`). Без приглашения никто другой ничего не увидит.

Назначение: **источник правды для совместной работы**. Когда второй разработчик берёт ведение проекта и сохраняет изменения — они идут сюда. Когда владелец заходит после паузы — он подтягивает свежее отсюда. Обе стороны видят одинаковую картину репозитория.

## backup — локальная резервная копия владельца

Адрес: `/Users/ilya/Backups/ai-dev-system.git` (на машине владельца, bare-репозиторий).

Видимость: только владелец. Это просто папка на его диске.

Назначение: **личная страховка от потери диска**. Если основной диск выходит из строя или папка с репозиторием случайно стёрта — историю можно восстановить отсюда. Не для совместной работы — второй разработчик к этой папке вообще не имеет доступа.

## Правило отправки

Скрипты ведения проекта (`scripts/take-shift.sh`, `scripts/release-shift.sh`, `scripts/approve-critical.sh`, `scripts/new-audit-task.sh`) отправляют свежие изменения **в обе копии**:

- `origin` — обязательно. Без этого второй разработчик не увидит изменения и может стартовать работу на устаревшем состоянии.
- `backup` — для владельца, локальная страховка.

Если `origin` недоступен (нет сети, GitHub упал) — скрипт показывает предупреждение, но `backup` всё равно обновляется, и работа продолжается локально. При следующем удачном вызове скрипт догонит `origin`.

Если `origin` отверг запись (например, кто-то другой опередил) — скрипт останавливается с ошибкой, потому что наш локальный стейт расходится с источником правды. Решение — подтянуть свежее (`git pull --rebase origin master`), при необходимости разрулить конфликт, попробовать снова.

## Что НЕ публикуется на GitHub

Astro остаётся локальным — продуктовый репозиторий `astro` живёт только на машине владельца, бэкап — в `/Users/ilya/Backups/astro.git`. На GitHub Astro не выкладываем. Это отдельное решение владельца, не имеет отношения к `ai-dev-system`.

Sitka-office — это продуктовый репозиторий, у него своя удалённая копия на GitHub (отдельно от `ai-dev-system`), куда уходит производственный код кассы.

## Если нужно изменить адрес или добавить новую копию

Это правка `policies/REMOTES.md` плюс ручная команда `git remote add` / `git remote set-url`. Делает владелец. Изменения git topology (новые удалённые копии, force-push, переписывание истории) — операции класса «обсуждаемые с владельцем перед действием».

## Связанные файлы

- `policies/USERS.md` — список разработчиков и их прав.
- `policies/SHIFTS.md` — правила ведения проекта (откуда вызываются скрипты, отправляющие в обе копии).
- `scripts/take-shift.sh`, `scripts/release-shift.sh`, `scripts/approve-critical.sh`, `scripts/new-audit-task.sh` — отправляют сюда.
