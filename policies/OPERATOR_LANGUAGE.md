# Операторский язык — главный список запрещённых слов

Этот файл — **главный источник правды** для слов, которые не должны попадать в адрес пользователя в каналах общения от агентов системы. `CLAUDE_GLOBAL.md` и `corrections/global-corrections.md` → Correction 013 содержат короткие подборки и ссылку сюда; полный список ведётся в этом файле.

## Зачем

Пользователь — оператор продукта, не разработчик и не DevOps. Англо-технический жаргон в его адрес ломает доверие и превращает систему в шум. Полный закон — `CLAUDE_GLOBAL.md` → «Главный закон общения с пользователем». BAD/GOOD примеры — Correction 013.

Этот файл нужен потому, что закон работает только при автоматической проверке. Скрипт `scripts/self-check-handoff.sh` читает список ниже и проверяет передачи (HANDOFF) на наличие этих слов в **user-facing зонах** перед отправкой задачи на приёмку.

## Где проверка применяется

Только внутри **user-facing зон** HANDOFF файла — блоков между маркерами:

```
<!-- user-facing -->
Текст, который пойдёт через TL пользователю (как есть или с минимальной обработкой)
<!-- /user-facing -->
```

Эти маркеры расставляет автор HANDOFF (Worker / Reviewer / TL) когда часть передачи содержит готовый текст для пользователя. Если маркеров в HANDOFF нет — проверка пропускается. Полный текст HANDOFF на внутреннем агентном языке — норма, ругаться нечего.

Внутри backticks слова разрешены — это технические имена (`Status`, `pytest`, `commit`), они нужны в технических разделах HANDOFF, не для пользователя. Скрипт игнорирует backtick-блоки при поиске.

## Что НЕ ловится автоматической проверкой

Смешанные русско-английские конструкции: «visual verdict», «code-side accept», «render clean», «smoke прошёл», «accept на ревью». Отдельные слова могут попасть в список, но контекст важнее — это работа автора при ручной самопроверке перед отправкой передачи.

## Категории и слова

Скрипт `scripts/self-check-handoff.sh` читает блок ниже как один плоский список — по одному токену на строку. Категории-заголовки (`# ...`) и пустые строки игнорируются. Сравнение case-insensitive по слову целиком (word boundary).

## Как добавить слово

1. Дописать в соответствующую категорию выше.
2. Если категория новая — добавить заголовок-комментарий (`# Категория`).
3. Прогнать `scripts/self-check-handoff.sh` на одном существующем HANDOFF — убедиться что нет ложных срабатываний.

## Связанные источники

- `CLAUDE_GLOBAL.md` → «Главный закон общения с пользователем» — концептуальный закон, короткий список ключевых групп.
- `corrections/global-corrections.md` → Correction 013 — BAD/GOOD примеры с пояснениями.
- `scripts/self-check-handoff.sh` — машинная проверка.
- Memory `feedback_operator_language.md` (личная для текущего агента) — обязательная самопроверка перед каждым сообщением пользователю.
