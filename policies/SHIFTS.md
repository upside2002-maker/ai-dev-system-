# Посменная работа главных по проекту

В команде два разработчика. Каждый может быть главным по проекту, но в один момент времени активный главный по конкретному проекту — один человек. Это нужно чтобы две параллельные сессии не редактировали один и тот же overlay проекта одновременно и не наступали друг другу на хвост.

## Что такое смена

Смена — это запись в файле `TL_SHIFT.md` внутри папки проекта в системе разработки (`project-overlays/<slug>/TL_SHIFT.md`). В ней:

- кто сейчас главный (email из `git config user.email`);
- когда взял смену;
- до какого времени взял (обычно текущее время плюс 8 часов);
- описание зоны работы одной фразой («разбор передачи от dm-7-c», «первый разбор задачи новой фазы»);
- ссылка на текущую задачу, если есть.

Когда смена свободна — в шапке файла стоит `Released: yes`, активного главного нет.

## Как взять смену

В терминале:

```
make take-shift SLUG=sitka-office SCOPE="разбор передачи"
```

Обычная длина смены — 8 часов. Если хочешь взять на другой срок:

```
make take-shift SLUG=sitka-office SCOPE="..." HOURS=4
make take-shift SLUG=astro SCOPE="..." HOURS=12
```

Что делает скрипт:

1. Подтягивает последние изменения из резервной копии — чтобы увидеть свежий `TL_SHIFT.md` и не работать на устаревшем.
2. Читает текущее состояние смены.
3. Если смена свободна или срок предыдущей истёк — записывает тебя как нового главного и сохраняет это в журнал репозитория, потом отправляет в резервную копию.
4. Если смена занята живым человеком и срок не истёк — отказывает с понятным сообщением «сейчас на смене такой-то, до такого-то времени».

Если кто-то опередил тебя и сохранил свою смену на полсекунды раньше — скрипт не сможет отправить изменения в резервную копию, скажет об этом и попросит пересинхронизироваться.

## Как освободить смену

```
make release-shift SLUG=sitka-office NOTES="что сделано за смену"
```

`NOTES` обязательно непустое. Это короткое описание что сделано за смену, пара фраз для второго разработчика. Скрипт проверит, что освобождаешь именно ты (не чужую смену), и обновит файл — поставит `Released: yes` и допишет заметки в раздел `## Notes`.

## Что такое «срок истёк»

Если человек взял смену 8 часов назад и не освободил — смена считается истёкшей. Любой может взять её на себя без отдельного разрешения. Это страховка от ситуации «взял и забыл».

Если поработал и забыл освободить — ничего страшного, новый главный просто запишет себя поверх. Желательно всё-таки освобождать руками — тогда заметки попадут в файл и второй разработчик увидит что было сделано.

## Если хочется работать на чужой активной смене

Не делай молча. Варианты:

- договориться с тем, кто на смене (через мессенджер, обычным человеческим способом — система этим не занимается);
- если очень срочно — попросить Илью прервать смену через специальный режим. Это **экстренное прерывание**, добавляется в следующем шаге плана. Пока недоступно.

## Если хочется только посмотреть, не править

`TL_SHIFT.md` не запрещает чтение. Чтение overlay и продуктового кода всегда свободно. Запись в overlay — только через смену.

## Active TASK в шапке

Поле `Active TASK: <путь>` в `TL_SHIFT.md` показывает текущую активную задачу главного. Обновляется руками — открой `TL_SHIFT.md` и впиши путь к файлу задачи когда переключаешься на неё. Это вспомогательная информация для второго разработчика, чтобы он видел над чем ты работаешь без необходимости спрашивать.

## Связанные файлы

- `policies/USERS.md` — список разработчиков и их прав.
- `scripts/take-shift.sh` — скрипт захвата смены.
- `scripts/release-shift.sh` — скрипт освобождения смены.
- `project-overlays/sitka-office/TL_SHIFT.md` — состояние смены для проекта sitka-office.
- `project-overlays/astro/TL_SHIFT.md` — состояние смены для проекта astro.
