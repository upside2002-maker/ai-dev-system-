# Пользователи системы разработки

Список людей, которые работают как главные по проектам, и какие у них права.

Файл читают скрипты `scripts/take-shift.sh`, `scripts/accept-task.sh`, `scripts/approve-critical.sh` — отсюда они узнают, кто имеет право прервать чужую смену и кто имеет право подписывать изменения в зоне обязательной подписи. Поэтому формат записи строгий: email на отдельной строке после имени, поля `can_*` — каждое на своей строке.

## Поля

- `email` — рабочий email. По нему скрипты различают одного человека от другого. Email берётся из `git config user.email` в момент действия.
- `can_override` — может ли прервать чужую активную смену главного по проекту. Сейчас только Илья как владелец.
- `can_approve_critical` — может ли подписывать изменения в зоне обязательной подписи (методологические файлы, описание ролей, политики, шаблоны, скрипты системы — полный список в `policies/CRITICAL_PATHS.md`).
- `can_deploy` — может ли выкатывать в продакшен.

## Правила формата

- Email пишется без backticks, в одну строку после `- email:`. Скрипты ищут по подстроке после двоеточия.
- Поле `can_*` — `yes` или `no`, всё нижним регистром.
- Не вставляй пустые строки внутри блока одного пользователя — скрипты читают блок до следующего заголовка `###`.
- Новый пользователь — новый блок `### Имя` с тем же набором полей.
