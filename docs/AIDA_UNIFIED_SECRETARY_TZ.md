## 0. Что строим

Один секретарь с тремя лицами — голос на маке («Аида»), приложение/браузер Claude (в т.ч. телефон), текст в Телеграме — которые читают и пишут ОДНО состояние из файлов под git. «Тот же секретарь» дают два РАЗНЫХ механизма (не путать):

1. **Удалённое управление (Remote Control)** — официальная фича Anthropic под подпиской: телефон/браузер/терминал — это ОКНА в ОДНУ живую сессию Claude Code, бегущую на маке. Бесшовность диалога комп↔телефон — отсюда. (Источник: `code.claude.com/docs/en/remote-control`, ledger F-20260612-011.)
2. **Единая файловая память (ledger + overlays)** — общий «мозг», из которого все лица восстанавливают одинаковое состояние одной командой. «Тот же секретарь по знанию хозяйства» — отсюда.

Голос — отдельный контур на платном движке OpenAI Realtime, он НЕ ходит на телефон через удалённое управление (с телефона — текстовая та же сессия, не голос). Голос выводится из круга сменой курса, а не добиванием (§3).

**Фундамент уже стоит (доказано файлами):** `ledger/*.jsonl` + `scripts/aida_ledger.py` (только дозапись — И-14; источник обязателен — И-5; карантin raw→promote — И-4; срок годности — И-13), в `make check`.

## 2. Единая память и регидратация (D.3): один мозг + окно как расходник

### 2.1. Decision к фиксации (нужно слово Owner'а, §6 #2)
Единый секретарь = одна файловая память; лица — читатели/писатели ОДНОГО состояния, не носители состояния; источник истины — `ledger` + git; прозовые снимки (CURRENT_STATE/STATUS_RU) — человеко-страница, не источник. Это меняет критпуть (И-11) → требует записанного решения Owner'а.

### 2.2. Единый слой `scripts/aida-brief` (корень всего)
Сейчас лица восстанавливаются по-разному и ни одно не читает реестр-ядро. Строим ОДИН слой, который все зовут одинаково. Собирает один экран из ИСТОЧНИКОВ: активные решения + факты (не-stale, с источником) + противоречия + счётчик карантина; live-снимки git через `scripts/snapshot.sh` (не из прозы — И-13); активные задачи/передачи; непрочитанный ящик. Два формата: `--for-agent` и `--for-owner` (операторский язык + источник в скобках).

### 2.3. ВАЖНАЯ ПОПРАВКА КРИТИКА — куда реально подключать
Живой голос — это `realtime_agent.py` (движок OpenAI), и он читает состояние через **`aida_status.py`** (`read_project_status`/`read_dispatcher_summary`) — ТОТ ЖЕ модуль, что и телефон. `brain.py::_read_holding_digest` — **старый модуль, его правка живой голос НЕ изменит.** Поэтому точек интеграции не три, а **две**:
- **D3.A — `aida_status.py`** переключить на `aida-brief` (это разом чинит И голос, И телефон); белый список и защиту от 14 атак НЕ ослаблять.
- **D3.B — `policies/SESSIONS.md`**: протокол «обнови контекст» → первая команда сессии `bash scripts/aida-brief --for-agent`.
`brain.py` — легаси, правится опционально и вторично.

### 2.4. Свежесть (поправка критика)
`aida-brief` обязан читать live на КАЖДЫЙ вызов; точка интеграции зовёт его НА КАЖДЫЙ вопрос, а не один раз при прогреве (иначе голос отдаёт закэшированное — тот самый рассинхрон И-13, который и чиним). `refresh_state()` в `brain.py` сейчас мёртв (не вызывается) — не опираться на него.

### 2.5. «Сам занимается окном» — ПОЛНЫЙ АВТОМАТ (требование Owner'а 13.06)
**Решение Owner'а:** ноль команд, Owner вообще не думает про контекст; и это должно быть НАДЁЖНО. Не «восстановление одной командой», а самоуправление без участия человека. Фундамент тот же (память в файлах), но добавляется автоматика:
- **Лица голос/телефон** уже авто-подтягивают состояние на КАЖДЫЙ заход через `aida-brief` (команд нет — встроено). Это и есть автомат для них.
- **Сессия за компом (Claude Code):** добавить авто-триггеры, чтобы Owner не набирал ничего: (а) хук старта сессии (`SessionStart`) сам зовёт `aida-brief --for-agent` — регидратация без команды; (б) при приближении к переполнению — авто-компакция (штатная у Claude Code) + авто-резюме этапа в `MAILBOX/archive`; (в) приоритет — НАДЁЖНОСТЬ: если авто-сворачивание не сработало, лицо честно говорит «состояние не прочитал» (И-3), а не врёт по памяти.
- Это отдельная задача поверх `aida-brief` (хуки + авто-резюме), запланирована как обязательная, не опциональная. Ручную дисциплину за систему не выдаём — строим именно автомат.

### 2.6. Запись голоса — только карантин (И-4, И-1)
Когда голос получит право писать — единственный разрешённый вызов `scripts/aida inbox '<заметка>' --context '...' --by voice`. Никаких `fact/decision/promote` с голоса. Сейчас боевой голос вообще не пишет (два инструмента, оба на чтение) — это ДОБАВЛЕНИЕ нового права; нужен «красный» тест лаборатории: «запиши факт голосом» → отказ с маршрутом в карантин.

### 2.7. Единый доклад
Список запрещённых слов вынести в общий файл, который читают и проверка передач, и `aida-brief`, и телефон (сейчас закон реализован трижды по-разному). Каждый факт — с коротким источником («прод на b1c9107 — проверено git, сегодня»).

## 5. Ворота качества
- Каждая задача: Worker (Opus) → независимый Reviewer (холодная сессия, И-12) → приёмка; код — тесты до Reviewer'а.
- `aida-brief` — единая точка отказа всех лиц: обязателен тест + деградация (ядро недоступно → лицо честно «состояние не прочитал», не отвечает из памяти модели — И-3); тест деградации покрывает ИМЕННО путь `aida_status`/живого голоса, не только сам слой.
- Голос — жёсткий гейт живого теста (изоляция врёт); регресс-чек всех прошлых дефектов.
- Правило двух кругов; защита от ложной полноты; сводку делает координатор сам.
