# Ядро правды v0.1 — как устроено и как пользоваться

Основание: D-20260614-003, D-20260614-004. ТЗ:
`docs/AIDA_TRUTH_KERNEL_V01_TZ_2026-06-14.md`. Только стандартная библиотека
python3 (как `scripts/aida_ledger.py`). Критпуть (`scripts/**`, `schemas/**`) —
изменение под подпись Owner'а.

## Что это

Детерминированный гейт перехода «мысль → опора». Ядро проверяет НЕ мысль, а
момент, когда модель **говорит факт / даёт совет / предлагает инструмент**. Тогда
вопрос: на чём держится? **Вердикт выносит КОД против доверенного источника, не
модель-судья.** Асимметрия: любая неопределённость → «не проверено»/«стоп»,
никогда «проверено».

## Анти-ведро (главное)

Модель НЕ подаёт произвольную команду-проверку. Она **выбирает** источник из
БЕЛОГО СПИСКА адаптеров и сообщает ожидаемое значение; код сам лезет в источник
и сверяет. Произвольной команды на исполнение в claim нет как класса.

Белый список (`aida-kernel adapters`):

| адаптер | источник | что отдаёт |
|---|---|---|
| `ledger.fact(key)` | `ledger/facts.jsonl` | значение активного факта по структурному ключу |
| `project.status(slug)` | `PROJECTS/projects.json` | есть проект в реестре или нет |
| `git.snapshot(repo)` | git | короткий HEAD реального репозитория |
| `capability.exists(name)` | `PROJECTS/capabilities.json` | статус возможности |
| `billing.manual_record(key)` | `ledger/facts.jsonl` | ручная запись Owner'а **со сроком годности** |
| `file.exists(path)` | диск (под корнем репо) | файл есть/нет |
| `test.result(test_id)` | `PROJECTS/test_results.json` | результат **заранее зарегистрированной** проверки |

Адаптер не из списка → claim «не проверено» (никогда «проверено»).

## Три столба

1. **Правда.** `type=fact` без адаптера-из-списка → «требует источника», наружу
   НЕ как факт, в память не пишется. Проверяемое: исполнить адаптер, сверить с
   ожидаемым — совпало → «проверено»; не совпало → «противоречит источнику»
   (стоп); адаптер упал/нет ожидаемого → «не проверено».
2. **Контекст.** Структурная память факта: `key`/`value`/`valid_until`/`source`
   (обратносовместимое расширение — старые факты без них валидны). Совет
   ДЕКЛАРИРУЕТ опоры (`supports`). Опора временная (`valid_until` есть) + горизонт
   совета постоянный (`horizon=permanent`) → стоп «опора временная, нельзя как
   постоянная основа без пометки». Противоречие памяти — **механически по key**
   (значение в памяти ≠ значению в claim → стоп). Без модели.
3. **Возможности.** Реестр `PROJECTS/capabilities.json`. Перед предложением
   внешнего инструмента — проверка `capability.exists`: если под ту же потребность
   (`covers`) есть **встроенное**, а предложен внешний костыль первым → стоп
   «сначала встроенное».

## Ворота выхода

- Наружу: статус-плашки. `passed` → `[проверено] …`; `unverified` →
  `[не проверено] …`; `stop` → ответ не выпускается, отказ с причиной.
- В память (`facts.jsonl` через `aida fact`): только «проверено».
- В `corrections/model_errors.jsonl` (append-only): автозапись при пойманной лжи.

## Как звать

```
# вердикт по claim (JSON из stdin/файла); код выхода: 0 passed, 3 unverified, 4 stop
echo '{"type":"fact","text":"кредит 200","adapter":"ledger.fact","arg":"api_credit","expected":"200"}' \
  | scripts/aida-kernel eval --json

# полные ворота выхода: вердикт + наружный текст + дозапись журнала ошибок
scripts/aida-kernel gate --file claim.json

# белый список адаптеров (проверка анти-ведра)
scripts/aida-kernel adapters
```

Схема claim — в шапке `scripts/aida_kernel.py`. Машинные коды вердиктов —
`REASONS` там же.

## Честный предел v0.1

Ядро ловит глюк/неуверенность/противоречие/просрочку и закрывает подделку
команды (адаптеры из белого списка). **НЕ ловит** правдоподобную ложь там, где
**сам доверенный источник** неполон/неверен (напр. Owner ошибся в ручной записи)
— это не вина ядра, но названо явно.

## Проверки

`make check` (через `make lab`) гоняет группу `tests/truth_kernel`: три красных
теста (200 временные → стоп; Телеграм при встроенном → стоп; факт без источника →
не проверено) + источник-врёт → стоп + невалидный claim → не проверено +
счастливый путь → проходит + анти-ведро (произвольная команда → не проверено) +
противоречие памяти → стоп + контракт белого списка + свободное рассуждение мимо
гейта.
