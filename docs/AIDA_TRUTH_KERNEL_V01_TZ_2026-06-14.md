# ТЗ: Ядро правды v0.1 (Truth + Context + Capability)

**Основание:** D-20260614-003, D-20260614-004. Класс: Tier A strict (критпуть scripts/**, schemas/** → подпись Owner на слияние). Исполнение: Worker → независимый Reviewer (оба Опус) → подпись Owner.

## Принцип (нерушимый)
Ядро проверяет НЕ мысль, а **переход от мысли к опоре**. Свободное рассуждение не трогаем. Гейт срабатывает, когда модель: **говорит факт / даёт совет / предлагает инструмент-действие**. Тогда вопрос: на чём держится?

**Вердикт выносит ДЕТЕРМИНИРОВАННЫЙ КОД против доверенного источника. НИКОГДА не модель-судья.** Асимметрия: любой глюк/неопределённость → «не проверено»/стоп, никогда «проверено».

## Анти-ведро (главное требование)
Модель НЕ даёт произвольную команду-проверку (иначе подделка через печать нужного = ведро со штампом «проверено»). Модель **выбирает** источник из БЕЛОГО СПИСКА адаптеров. Список v0.1:
- `ledger.fact(key)` — структурный факт из памяти;
- `project.status(slug)` — статус проекта (реестр + снимок);
- `git.snapshot(repo)` — реальное состояние репозитория;
- `capability.exists(name)` — есть ли встроенная возможность;
- `billing.manual_record(key)` — ручная запись Owner'а (подписка/кредит) **со сроком годности**;
- `file.exists(path)` — файл реально есть;
- `test.result(test_id)` — только заранее зарегистрированная проверка.

Адаптер с неизвестным именем / не из списка → claim «не проверено» (никогда «проверено»).

## Что строим — три столба

### 1. Правда (Truth)
- `type=fact` без источника-из-белого-списка → статус «требует источника», наружу НЕ как факт, в память не пишется.
- Проверяемое → исполнить адаптер, сверить с ожидаемым: совпало → «проверено»; не совпало → «противоречит источнику» (стоп); адаптер упал/нет ожидаемого → «не проверено».

### 2. Контекст (Context) — закрывает случай «200 временные»
- **Структурная память**: факт несёт `key`, `value`, `valid_until`, `source` (обратносовместимое расширение схемы факта — добавить опциональные поля, старые факты валидны).
- claim/совет ДЕКЛАРИРУЕТ опоры (на какие факты опирается). Если опора временная (`valid_until` есть) и горизонт совета её превышает / совет про «постоянный процесс» → **стоп**: «опора временная (до такого-то), нельзя как постоянная основа без пометки».
- Противоречие с памятью — **механически по key**: значение факта в памяти ≠ значению в claim → «противоречит памяти» (стоп). Без модели.

### 3. Возможности (Capability) — закрывает случай «Телеграм»
- Новый реестр `PROJECTS/capabilities.json` (как projects.json): `name`, `status` ∈ {встроено / внешнее / нужен-апи / нужна-подписка / нужна-настройка / костыль}, `covers` (какую потребность закрывает).
- Перед предложением инструмента (`type=recommendation`/`action` с инструментом): проверка `capability.exists` — есть ли **встроенное**, покрывающее ту же потребность. Если есть, а предложен внешний костыль первым → **стоп**: «сначала встроенное».

## Ворота выхода
- Наружу: ядро пересобирает черновик, подставляя статус-плашки; факт не «проверено» → «не проверено: …»; при стопе ответ не выпускается — отказ с причиной.
- В память (`facts.jsonl` через `aida fact`): только «проверено», с заполненными key/value/valid_until/source.
- В `corrections/model_errors.jsonl` (новый, append-only): автозапись при пойманной лжи (противоречит источнику/памяти / без источника / устарело).

## Скорость (анти-тормоз)
Гейт только на несущем (факт/совет/инструмент/действие/память/деньги). Обычный разговор — мимо. Проверки детерминированные миллисекундные. Второго прохода моделью НЕТ.

## Переиспользовать (не дублировать)
И-5 (`_build_source`/`SOURCE_KINDS`), И-13 (`EXPIRES_RE`), И-14 (`contradictions.jsonl` — расширить на «сторона-claim», т.к. claim не факт), `ledger_lock`/`allocate_and_append`, контракт `run_check`, образец асимметрии из `crypto-platform/.../classify.py` (выход модели через схему, иначе отброс), тест-каркас `evals/lab/lib/harness.sh`. stdlib-only.

## ТРИ КРАСНЫХ ТЕСТА (жёсткий гейт приёмки)
1. **200 временные → стоп.** Память: `api_credit{value:200, valid_until:дата}`, `subscription{autorenew:off}`. Совет «строить постоянный мост через апи/Телеграм» → стоп «ресурс временный, не постоянная основа». Поймано детерминированно.
2. **Телеграм при встроенном → стоп.** Реестр: `claude.remote_control=встроено`, `claude.channels=встроено`, `telegram_bridge=внешнее`. Совет «Телеграм первым» → стоп «сначала встроенное».
3. **Факт без источника → «не проверено».** Нет записи в доверенном источнике → наружу только «не проверено», в `facts.jsonl` не пишется, строка в `model_errors.jsonl`.
Плюс: факт-источник-врёт → стоп; невалидный claim → «не проверено»; счастливый путь (валидный + проверенный) → проходит.

## Честный предел (вписать в приёмку чёрным по белому)
v0.1 ловит глюк/неуверенность/противоречие/просрочку. Закрыт и случай подделки команды (адаптеры из белого списка, произвольных команд нет). НЕ ловит: правдоподобную ложь там, где **сам доверенный источник** неполон/неверен (напр. Owner ошибся в ручной записи) — это не вина ядра, но надо назвать.

## Процесс
Worker в изолированной копии, ветка от master, коммит с трейлером `Approved-by: upside2002@gmail.com` (Owner направил), НЕ сливать. make check зелёный. Reviewer (холодный) бьёт по «ведру»: нет ли пути произвольной команды, выносит ли вердикт код а не модель, проходят ли 3 красных теста детерминированно, не гейтится ли обычный разговор. Затем — подпись Owner на слияние.
