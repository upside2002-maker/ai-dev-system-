## 0. Противоречие и как оно снимается

Owner хочет двух вещей сразу: загрузить окно по максимуму И не уронить качество (деньги, клиенты, прод, реклама, экспертная астрология — высший риск).

Снимается **не отказом от объёма, а дисциплиной ворот.** Объём масштабируется на read-only слое, где цена ошибки = неверное число на экране. Высокий риск (класс A) — узкое горло по конструкции: независимый Reviewer холодной сессией (И-12) + записанное решение Owner'а; пропускаются по одному.

## 1. Честный потолок загрузки (снизу, а не сверху)

Реальный потолок ставит не число агентов и не котёл Max20, а **число вдумчивых независимых Reviewer-сессий, которые Owner реально вывезет за окно**, минус те, что уже заняты.

Поправка критика принята: **живое денежное ядро крипты уже занимает tier-A Reviewer прямо сейчас** (см. §2). Поэтому честный потолок:

- **Сейчас: 2–4 качественных read-only пары** (Worker+Reviewer на Opus), пока ядро крипты держит tier-A проверяющего и один тяжёлый прогон/окно занят.
- **После разблокировки (волна 2):** широкий read-only фронт крипто-наполнения масштабируется (агент-на-видео), но это считается как **один тяжёлый прогон на окно**, с обязательной сверкой полноты, а не как бесплатный параллелизм.
- **Класс A — никогда валом.** По одному, с независимым Reviewer'ом и записанным «да» Owner'а.

## 5. Дисциплина качества (сердце плана)

- **Ворота на каждой задаче:** Worker (Opus) → независимый Reviewer (Opus, холодная сессия) → приёмка; код — тесты зелёные до Reviewer'а.
- **Класс A:** Reviewer независим по конструкции (И-12) + записанное «да» Owner'а + «красный» тест (опасное без подтверждения → блок).
- **Голос — жёсткий гейт (поправка критика):** Phase 2.x принимается ТОЛЬКО по живому тесту Owner'а. Изоляционные тесты голоса систематически врут — «зелёнка» без живого теста запрещена. Это инвариант ветки D.2, не мягкий вопрос.
- **Правило двух кругов:** третий заход «шаг вперёд–шаг назад» — только классом выше или сменой подхода, с решением Owner'а (беда стоп-слова голоса — прямой кандидат).
- **Защита от ложной полноты:** перед приёмкой любой пачки координатор сверяет план-факт по единицам, читает вывод критика (не только зелёный флаг), из упавшего прогона сперва вычитывает частичные данные. **Для крипто-наполнения (≈105 видео) обязателен НАЗНАЧЕННЫЙ механизм сверки источника на каждую единицу** (§6 #0-б) — без него «агент-на-видео» = фабрика правдоподобного мусора.
- **Один тяжёлый прогон на окно.** Претендентов сейчас ТРИ (поправка критика): ревью ядра Т-3b, крипто-извлечение, астро-извлечение Дарагана. В одно окно — один. Координатор разводит, синтез делает сам.
- **Маркировка факт/гипотеза** с источником (И-3, И-5); астро-правила draft→approved Мариной.
- **Операторский язык** во всех докладах Owner'у/в каналы/Марине.

## 7. Чего НЕ делать

- Не грузить валом — объём только там, где Reviewer на каждую пару.
- Не пропускать ревью; зелёный итог многоагентного прогона ≠ полный охват.
- Не запускать высокориск-агентов без стадий и подтверждения (деньги/клиенты/реклама/прод).
- Не выдавать декларацию за систему (пустой скилл, ненатёртый mcp, код голоса ≠ сданный голос).
- Не подавать браузерное расширение как 24/7-автономность (полу-ручное на живой сессии).
- Не ставить два тяжёлых прогона в одно окно (теперь претендентов три — ядро тоже).
- Не лечить предел заплатой на третий круг.
- Не давать голосу/агенту писать в проверенную память — только в карантин (И-4).
- Не трогать боевой продукт из смежной колеи; крипто-наполнение не правит денежное ядро.
- Не тащить PII в корпус/логи (golden-карты Астро; приватные дашборды крипты).
