# ТЗ: Структурные заявки (5 типов) — фронт поверх влитых ядер

**Основание:** D-20260614-007. Вердикт дизайн-ревью: годен с правками. Класс: Tier A strict
(критпуть scripts/**, schemas/**) → подпись Owner на слияние. Worker → независимый Reviewer → подпись. stdlib only.

## Принцип
Ворота срабатывают НЕ на свободный текст, а на МОМЕНТ, когда текст становится опорой.
Этот момент модель объявляет САМА, выставляя заявку-конверт. До объявления речь свободна.
Грубый сторож свободного текста ОТКЛОНЁН (ложные блоки) — НЕ воскрешать.

## Новое — РОВНО два файла (вся проверка — существующая, import не копия)
1. **schemas/claim.schema.json** (~60 строк) — конверт. Общие поля `kind`/`text`/`by`;
   по типу (oneOf, additionalProperties:false): невалидный/неполный → unverified (асимметрия).
2. **scripts/aida_claim.py** (~150 строк) — диспетчер: таблица 5 строк kind→переименование полей→
   вызов УЖЕ ВЛИТОГО верификатора. Сам НЕ проверяет. + CLI `aida claim eval|gate` (коды 0/3/4).

## Пять типов → маршрут в влитое
| kind | что | верификатор (влитой) |
|---|---|---|
| `fact_claim` | утверждаю факт | aida_kernel.gate → источник(7 адаптеров)+противоречие памяти |
| `advice_basis` | совет + опоры/горизонт | aida_kernel.gate → контекст-срок (временная опора как постоянная → стоп) |
| `capability_check` | предлагаю инструмент | aida_kernel.gate → реестр (встроенное под need → стоп) |
| `action_request` | меняю мир | aida_ledger.cmd_action (requested→executed, защиты влиты) |
| `done_report` | «сделано», сверьте | (а) run_check сверка-с-миром; (б) gate(fact) через file/git/test |

## Выставление — ДИСЦИПЛИНА, не парсер
Протокол модели: свободный текст как есть; в точке «утверждаю/советую/прошу-изменение/предлагаю-
инструмент/рапортую-сделано» — сначала эмитнуть конверт, прогнать через ворота, наружу при passed.
Безопасные авто-проверки — ТОЛЬКО на самой заявке (схема, полнота, белые списки), НИКОГДА на речи.

## ОБЯЗАТЕЛЬНЫЕ ПРАВКИ (критик, перед приёмкой)
1. **done_report ветка (а) — НЕ переоценивать.** `run_check()` глобальна (журнал↔мир целиком), а не
   проверка конкретного рапорта; ловит ложь, только если она уже в журнале как расхождение. Чисто
   словесное «готово» без выставленного действия ветка (а) НЕ видит — вписать прямо в ТЗ.
2. **Тест «executed-заявка есть, маркера в мире нет» (ledger ~1238-1243) — единственный прямой ловец
   лживого done_report — сейчас БЕЗ покрытия.** Красный тест #5 сделать обязательным, бьющим именно
   эту ветку (первый её зелёный прогон).
3. **Протокол выставления — закрепить как блокирующее решение,** не опцию (без него «главная дыра» шире).
   Рекомендация: `policies/` (долговечно, в репо) + короткий указатель в `CLAUDE.md`.

## Честный предел (в приёмку, чёрным по белому)
Необъявленная словесная опора — **вне охвата ПО КОНСТРУКЦИИ** (цена отказа от парсера речи).
Подстраховано постфактум: ворота дел ловят дело-мимо-заявки (обход журнала), память ловит повторное
противоречие по key, журнал лжи копит пойманное, ревью сверяет речь с журналом дел. Остаётся:
чисто словесная разовая ложь; неверный сам доверенный источник; слоя approved/rejected пока нет.

## Красные тесты приёмки (10)
fact-источник-врёт→стоп; advice временная-опора-как-постоянная→стоп (200); capability внешний-при-
встроенном→стоп (Телеграм); action честный→executed+маркер==change; **done «сделано»-а-мир-пуст→инцидент
(#5, обязателен)**; done без способа сверки→unverified; fact без adapter→unverified; reasoning/речь→мимо
(анти-тормоз); неизвестный adapter/act_kind→unverified/refuse (анти-ведро); невалидный конверт→unverified.

## Переиспользовать (не дублировать)
evaluate/gate, 4× _check_*, 7 адаптеров, aida_capabilities, cmd_action+защиты, run_check,
record_model_error, harness.sh, env-подмена путей. Ядра слов и дел НЕ трогать (остаются зелёными).
