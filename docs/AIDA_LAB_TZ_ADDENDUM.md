## §A. Починить лазейку: заглавная кириллица в evidence-фильтре (RED, критпуть)

**Дыра (подтверждена состязательно):** `scripts/self-check-handoff.sh` ловит
git-слово рядом с датой через `grep -iE '…|[Вв]ыкат|[Зз]адеплоен|…'`. Шаблоны
удваивают регистр ТОЛЬКО первой буквы, остальные хардкод-строчные, а `grep -i`
не складывает регистр кириллицы в локали C (скрипт идёт под `env -i LC_ALL=C`).
Итог: `ВЫКАТ … 2026-06-10` (всё заглавными) проходит БЕЗ предупреждения, тогда
как `commit/DEPLOY` (латиница) ловятся всегда.

**Чинить локале-независимо.** НЕ полагаться на `grep -i` для кириллицы. Способ
на выбор Worker'а, любой надёжный в C-локали:
- нормализовать кириллицу строки к нижнему регистру перед сравнением
  (`python3` stdlib: `.lower()` — он складывает кириллицу независимо от локали;
  можно прогнать поток дат-строк через короткую python-вставку), либо
- расширить классы на полный регистр каждой буквы ключа (`[ВвВ]` недостаточно —
  нужно покрыть ВСЕ буквы слова: `ВЫКАТ`, `Выкат`, `выкат`, смешанные).
  Первый вариант (python .lower()) надёжнее и короче — предпочесть его.

**Не сломать** существующее: латинские ключи, строчная кириллица, и
анти-ложный-срабат (голая дата прогноза `2048-03-01` без git-слова — без WARN)
обязаны работать как раньше.

**Тесты (добавить в `evals/lab/tests/self_check/`):**
- `selfcheck-evidence-uppercase-cyrillic-warns` (RED): строка ниже 15-й
  `ВЫКАТ НА ПРОД 2026-06-10` → WARN присутствует. Прогон под `env -i LC_ALL=C LANG=C`.
- параметрически или отдельными: `ЗАДЕПЛОЕН`, `СМЕРЖЕН` заглавными + дата → WARN.
- регресс-pass: `Прогноз транзита на 2048-03-01` → WARN нет (не сломали).
