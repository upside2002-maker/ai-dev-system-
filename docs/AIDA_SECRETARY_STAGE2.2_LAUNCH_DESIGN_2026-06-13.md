# Этап 2.2 — Запуск работы голосом. Дизайн + ТЗ + красная команда

**Дата:** 2026-06-13. **Источник:** многоагентный прогон `secretary-launch-work-design`.
**Статус:** дизайн готов; вердикт красной команды — **ТРЕБУЕТ ПЕРЕРАБОТКИ перед сборкой**.
**Гейт:** живой тест базового голоса 2.1 (ключ «Аида» + стоп) ещё НЕ сдан Owner'ом.

---

## Где мы сейчас

**Телефон — работает.** Удалённое управление Claude Code — это окно в ОДНУ живую
сессию на маке, у которой есть руки: с телефона команда «запусти воркера» реально
поднимает агента. Проверено живьём 13.06.

**Голос — не работает на запуск.** Живой голос Аиды — отдельный контур на платном
движке (OpenAI Realtime). У него ровно два инструмента, оба только чтение
(`get_project_status`, `get_dispatcher_summary` → зовут `scripts/aida-brief`).
Запустить воркера он не может.

## Настоящая дыра

Не «у голоса нет рук» — рук у него и НЕ должно быть (И-1). Дыра: между голосовым
контуром и живой сессией Claude Code **нет канала**. Телефон умеет, потому что он
буквально окно В живую сессию; голос — посторонний процесс рядом.

## Архитектура (принята как верная): Вариант Б — очередь через карантин, мозг за воротами

Голос пишет запрос ТОЛЬКО в карантин (`ledger/inbox.jsonl`, статус `raw` — единственное
право записи голоса). Решение «запускать/нет», подтверждение опасного, сам запуск и
приёмку делает живая сессия Claude Code (тот же мозг, что у телефона) за своими воротами.
Голос даже на «да, запускай» сам не запускает — лишь дописывает подтверждение в очередь.

Почему так: единственный путь, не рвущий И-1; один журнал и одни ворота (И-2/И-10);
переиспользует уже работающий запуск воркеров; запущенная работа наследует контур
качества Worker→Reviewer→приёмка — секретарь к продукту не прикасается.

## Поток команды

1. «Аида …» (без кнопок/наушников) → фраза в сессию.
2. Распознавание; имя проекта — по словарю синонимов, неизвестный → переспрос.
3. Инструмент `request_work(project, task, risk_hint)` пишет ОДНУ строку в inbox
   (`recorded_by=voice`, `context=voice:request_work …`), возвращает короткий ok-текст.
4. Голос сразу: «Принял — кладу в работу, доложу». Честно: запрос ПРИНЯТ, не «сделано».
5. Живая сессия подхватывает запрос (поллер/наблюдатель очереди).
6. Ворота + подтверждение: внутренняя задача — одно «да»; опасное (деньги/прод/наружу/
   удаление) — отдельная подтверждающая фраза по конкретному действию (И-7, не наследуется).
7. Запуск воркера штатным путём (`new-task.sh`), запись в журнал.
8. Worker → независимый Reviewer (холодная сессия, И-12) → приёмка с подписью.
9. Уведомление: мозг дописывает `brain:result` в очередь; голос озвучивает итог
   (активный поллер) либо отдаёт при следующем вопросе.

## ОБЯЗАТЕЛЬНЫЕ ПРАВКИ до сборки (красная команда)

1. **И-3 — закрыть по существу.** «Локальная whisper-расшифровка как истина для
   действий» в коде ОТСУТСТВУЕТ (то, что есть — серверная транскрипция той же платной
   модели). Аргументы `request_work` сейчас пришли бы из слуха модели — ровно то, что
   запрещено. Решение: **голосовая эхо-сверка СУТИ задачи перед записью** («Кладу:
   доводка парсера Ситки. Да?»), а не только подтверждение факта запуска. (Полный
   локальный отдельный STT — опционально, тяжелее; эхо-сверка обязательна.)
2. **Защита от двойника.** `next_id` в `aida_ledger.py:121` читает max+1 без блокировки —
   гонка при параллельной записи. Плюс нет дедупа: повтор команды → два воркера. Нужны
   блокировка `next_id` + ключ дедупа (проект+суть+окно времени).
3. **Таймаут ожидания результата.** Нет heartbeat: если мозг/воркер умер до записи
   `failed`, очередь молчит вечно. Нужен срок: нет `brain:result` за N минут → голос
   честно «по X доклада нет, статус неизвестен» (И-3), а не тишина. (Память: ложная полнота.)
4. **Подтверждение с привязкой.** `voice:confirm` должен нести `ref` конкретной строки
   запроса, иначе «да» приклеится не к тому запросу (И-7).
5. **Тяжёлый/денежный запуск.** Запретить одновременный тяжёлый многоагентный прогон с
   удалёнкой (оба занимают claude.ai/code — разорвут живую сессию). Денежное — вне И-8,
   только отдельным подтверждением.

## Критерии приёмки (после правок)

- Манифест голоса — ровно ТРИ инструмента; четвёртое имя → честный отказ + строка в журнал.
- `request_work` с известным проектом → одна корректная строка inbox + ok-текст; неизвестный
  → отказ, в очередь ничего.
- На команду — «принял, доложу» мгновенно (без ожидания запуска).
- Голос сам воркера НЕ стартует: «Аида, сама поправь файл» → отказ; в коде голоса нет
  Write/Edit/Bash сверх записи в inbox (проверка грепом).
- Эхо-сверка сути перед записью; подтверждение опасного — отдельной фразой (И-7).
- Дедуп: повтор команды не плодит второго воркера.
- Таймаут: нет результата за N минут → честное «доклада нет».
- Сквозной живой прогон по слову Owner'а: команда → inbox → мозг подхватил → воркер →
  Reviewer → приёмка → `brain:result` → голос озвучил. Сверить inbox с фактическим запуском.
- Честная деградация при недоступной очереди/неподнятой сессии: «запрос не положил / мозг
  не на связи», не «запустил».
- Прежние тесты 2.1 зелёные.
