# Лаборатория проверок ядра Aida — как построено (evals/lab)

**Дата:** 2026-06-12
**Заказ:** Owner («го» по плану стройки). **Архитектор:** Admin.
**Сборка:** многоагентный рабочий процесс (карта ворот → синтез → сборка →
состязательная проверка → починка → критик) + добивка по находкам критика
(`docs/AIDA_LAB_TZ_ADDENDUM.md`). Все этапы — исполнители на Опусе, приёмка — Admin.
**Назначение:** превратить инварианты конституции `policies/AIDA_INVARIANTS.md`
из закона-для-людей в исполняемые тесты. Лаборатория на каждом прогоне доказывает,
что ворота реально БЛОКИРУЮТ нарушение (а не просто заполнены поля), что
легитимное проходит (анти-ложный-срабат, защита И-15), и что пути методологии
существуют (контракт-тесты).

## Где живёт

- `evals/lab/run.sh` — единый раннер; `make lab` — алиас; включён в `make check`.
- `evals/lab/lib/harness.sh` — хелперы (assert, временные репо/реестры/транскрипты).
- `evals/lab/tests/` — тесты по группам: accept_task, commit_msg, self_check,
  ledger, ru_guard, prepush, contract.
- `evals/lab/fixtures/` — шаблоны и список контракт-путей.

**Железное правило:** лаборатория НИКОГДА не пишет в живой репозиторий.
Любой мутирующий тест работает на временной копии (`mktemp -d`, `trap rm`).
Живые `ledger/`, `project-overlays/`, `.git/` побайтово нетронуты (проверено
приёмкой: md5 до/после совпадают).

## Укрепления живых ворот (побочный, ценный результат)

Состязательная проверка нашла и починка закрыла четыре реальных лазейки в
самих воротах (с тестами на каждую):
1. `accept-task.sh`: строчный `tier: a` проскакивал мимо всех трёх ворот Tier A
   на денежных задачах — нормализация регистра (`toupper`).
2. `accept-task.sh`: кандидат-HANDOFF без строки статуса ронял скрипт пустым
   кодом вместо внятного отказа — `|| true`.
3. `self-check-handoff.sh`: чисто десятичное число маскировалось под git-хеш —
   требуется hex-буква.
4. `self-check-handoff.sh`: заглавная кириллица (`ВЫКАТ`) обходила evidence-фильтр
   (`grep -i` не складывает кириллицу в C-локали) — починено локале-независимо
   через `python3 .lower()`; маркер зоны устойчив к пробелам и тексту на строке.

## Что осознанно вне покрытия (honor_system, печатается на каждом прогоне)

Поведенческие инварианты без программного слоя: И-1 (секретарь без рук),
И-2/И-3 (мир/истина только через ядро — ядра действий ещё нет), И-7 (опасное
с подтверждением — нет журнала действий), И-9 (изоляция доменов), И-10
(журнал действий — не существует, конституция это признаёт), И-15
(анти-бюрократия — процессный). И-8 (деньги) — отдельный проект crypto.
Смешанные ру-en фразы — фразовый контекст, на авторе. Авто-скан путей в доках —
следующая итерация (пока контракт ведётся явной фикстурой). Эти зоны закроются
по мере постройки соответствующих слоёв ядра — лаборатория растёт вместе с ним.
