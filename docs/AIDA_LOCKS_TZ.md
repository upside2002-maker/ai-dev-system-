## Замок 1 — Tier A: независимая проверка по файлу-доказательству (И-12, X-1)

**Сейчас.** `accept-task.sh` проверяет поле `Reviewer:` в шапке TASK (отказ на
inline/self/TL/пусто). Но поле можно заполнить, не проводя проверку — инцидент
X-1 (денежная математика принята без независимого Reviewer'а) прошёл именно так.

**Надо.** При приёмке TASK с `Risk tier: A` дополнительно требовать
**существующий Reviewer-HANDOFF**: файл в `project-overlays/<slug>/HANDOFFS/`
(или его `archive/`), у которого:
- `From:` — Reviewer (или `Role mode: Reviewer`);
- `TASK:` — ссылка на принимаемый TASK (по имени файла);
- `Status:` — не `open` (acknowledged или closed: проверка дошла до адресата).

Нет такого файла → отказ с русским операторским сообщением: что отсутствует и
как исправить (провести независимую проверку, оформить Reviewer-HANDOFF).
Найден → путь к нему печатается в выводе приёмки (след в журнал).

**Не трогать:** поведение Tier B/C.

**Приёмка (фикстуры во временной папке, не в репо):**
1. Tier A, всё как раньше требовалось, но Reviewer-HANDOFF'а нет → отказ.
2. Есть HANDOFF по той же задаче, но `From: Worker` → отказ.
3. Корректный Reviewer-HANDOFF (`From: Reviewer`, `TASK:` совпадает,
   `Status: closed`) → принимается.
4. Tier B без Reviewer-HANDOFF → принимается как раньше.

## Замок 2 — `make snapshot`: снимок состояния из источника (И-13)

**Сейчас.** Строки снимков («сервер на …», «HEAD …») в CURRENT_STATE и
DISPATCHER пишутся руками по памяти — класс ошибок «рассинхрон 0 при реальном
дрейфе» (обзор, ADS-DRIFT-1).

**Надо.** Новый `scripts/snapshot.sh` + цель `make snapshot SLUG=<slug>`:
печатает каноническую строку снимка продукт-репозитория проекта:
короткий HEAD, ветка, дата-время, чистота дерева (чисто / N неучтённых /
M изменённых), последний коммит одной строкой. Путь к продукт-репо определять
тем же способом, что `check-overlay-consistency.sh`. Работает и для
`pre-phase0` (репо известен). Только печать — никаких правок файлов.

**Приёмка:**
1. `make snapshot SLUG=sitka-office` → строка с реальным текущим HEAD.
2. Несуществующий slug → внятный отказ.

## Замок 3 — подпись критпутей в коммите + точный evidence-фильтр (И-11)

**3а. Подпись критпутей.** В `.githooks/` сейчас есть pre-push (работает,
hooksPath включён). Добавить `pre-commit` (или аккуратно дополнить, если файл
уже существует — не затирая существующее поведение): если в индексе есть файлы
из `policies/**`, `scripts/**`, `templates/**`, `corrections/**` — требовать в
сообщении коммита трейлер `Approved-by: <email>`, где email имеет
`can_approve_critical: yes` в `policies/USERS.md`. Важно: список одобряющих
читать из **HEAD-версии** `USERS.md` (не из индекса) — чтобы нельзя было одним
коммитом вписать себя и самоодобриться. Нет трейлера / email без права → блок
с русским сообщением: какие файлы критичные, какой трейлер добавить.
(Сообщение коммита в pre-commit ещё недоступно стандартно — если по месту
правильнее `commit-msg` хук, сделать `commit-msg`; решение объяснить в отчёте.)

**3б. Evidence-фильтр.** В `self-check-handoff.sh` проверка «дата без хеша»
срабатывает на любую дату — на астро-передачах это дало 24 ложных
предупреждения (легитимные даты прогнозов), приучая игнорировать проверку.
Сузить: дата флагуется только если в той же строке есть контекст изменений —
`PR #`, `commit`, `merge`, `deploy`/`деплой`/`выкат`, либо хеш-подобное слово.
Голые даты (прогнозы, сроки, дни рождения) — не флагуются.

**Приёмка:**
1. Коммит с правкой файла в `policies/` без трейлера → блок; с
   `Approved-by: upside2002@gmail.com` → проходит.
2. Передача с «прогноз на 2048-03-01» → 0 предупреждений; с «PR #74
   (2026-05-01)» без хеша → предупреждение остаётся.

- Сообщения отказов — по-русски, операторские, с «как исправить».

  `scripts/self-check-handoff.sh`, `.githooks/pre-commit` или `commit-msg`
  (новый/дополнение), `Makefile` (только цель snapshot). Фикстуры — во
  временной папке.

  вопрос в отчёте.
