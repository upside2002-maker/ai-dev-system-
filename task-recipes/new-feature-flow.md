# New Feature Flow

## Когда использовать

Когда в продукт нужно добавить новую функцию без размывания архитектуры.

## Порядок

1. Определить, в каком слое живет истина о новой функции.
2. Проверить, не является ли задача на самом деле тремя отдельными задачами:
   core contract, service wiring, UI workflow.
3. Сначала зафиксировать форму данных и границы.
4. Потом менять ровно тот слой, который обязан владеть этой логикой.
5. Только после этого подключать соседние слои.

## На что смотреть

- если функция про деньги/статусы/инварианты: начинаем с core;
- если функция про сбор данных извне: начинаем с services;
- если функция про ускорение работы менеджера: начинаем с UI;
- если функция затрагивает все слои: дробим по ownership, а не пишем "сквозную магию".
